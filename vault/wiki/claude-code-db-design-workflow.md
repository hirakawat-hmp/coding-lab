# Claude Code とのペア DB 設計ワークフロー

TechTrain の著者 chocludy が実践する、Claude Code をペアパートナーとしてデータベース設計を進める 4 ステップワークフロー（2026年4月24日公開、Zenn・はてブ人気記事）。

## 前提

- コード生成・レビュー段階の 9 割以上を AI に委任
- しかし**設計段階は人間が主体**という思想
- Claude Code は「判断検証を加速するツール」として位置づける

## 4 ステップワークフロー

### ステップ 1：ユースケース列挙（AI 主導）

Claude Code が既存コードベースをスキャンし、想定されるユースケースの叩き台を生成。人間は「抜け漏れの確認」に集中できる。

### ステップ 2：ブラッシュアップ（人間主導）

人間が担う 2 つの役割：
1. **論点の結合・分離**：粒度が粗すぎ/細かすぎるユースケースを調整
2. **ドメイン知識注入**：業界慣行・法規制・暗黙知を追加

AI は「コードの文脈」は知っているが「ビジネスの文脈」は知らない。

### ステップ 3：各問いの壁打ち（AI との対話）

設計の各決定ポイントで Claude Code と議論。

- 「この関連テーブルは正規化すべきか？」
- 「ここで N+1 問題が起きないか？」

議論を通じて仕様の穴が発覚することを「歓迎」する姿勢が重要。

### ステップ 4：Subagent レビュー（マルチ AI 検証）

異なるペルソナを持つ **4 つの AI** が設計を多角的に検証：

| ペルソナ | 視点 |
|----------|------|
| パフォーマンスエンジニア | クエリ性能・インデックス |
| セキュリティエンジニア | アクセス制御・データ保護 |
| ドメインエキスパート | ビジネスルール整合性 |
| DBA | 整合性・正規化 |

## 核心的な気づき

> AI は判断検証を加速するツール。判断の主体は人間であるべき。

設計フェーズに AI を投入すると「答えが出る」のではなく、「問いが増える」ことが多い。これは悪いことではなく、設計品質向上のシグナルとして捉えるべき。

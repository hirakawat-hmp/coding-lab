# Backblaze がクラウド同期フォルダのバックアップを静かに停止した件

Backblaze が OneDrive・Dropbox・Git フォルダなどのクラウドストレージフォルダを静かにバックアップ対象外にした。告知なしのサービス縮小として HN で 928 ポイントを集め大きな議論を呼んだ。

## 何が起きたか

- **変更内容**: OneDrive・Dropbox・Git フォルダのバックアップを停止
- **告知方法**: リリースノートの「改善点」として埋め込むだけで、ユーザーへの直接通知なし
- **発見経緯**: ユーザーが 383GB の OneDrive データがバックアップされていないと気づいた

## Backblaze の公式見解

「クラウドストレージプロバイダーによるパフォーマンス問題・過剰なデータ使用・意図しないアップロードを防ぐため」と説明。

## 問題点

### サービスと現実のギャップ

2015 年当時の約束: 「デフォルトですべてのユーザーデータを含む」「ファイルタイプやサイズの制限なし」

現在: クラウド同期フォルダは対象外（でも公式ドキュメントに明記なし）

### クラウド同期 ≠ バックアップ問題

| サービス | 保持期間 |
|----------|---------|
| OneDrive / Dropbox | 30日（デフォルト） |
| Backblaze | 1年 |

OneDrive と Dropbox はバックアップではなく同期サービスであり、誤削除や ransomware からの長期保護にはならない。Backblaze がこれを除外することで**ユーザーのデータが無保護になるリスク**がある。

## 教訓

1. **バックアップの範囲を定期確認**: サービスが静かに変更されても気づけない
2. **同期 ≠ バックアップ**: クラウドストレージへの依存は 3-2-1 ルールを満たさない
3. **サービス変更の通知方法を確認**: 重要な縮小はリリースノートではなく直接通知されるべき

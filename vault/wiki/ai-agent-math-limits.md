# AIエージェントの数学的限界：「エージェント元年」はいつ来るのか

![AI Agents - Math Doesn't Add Up](https://media.wired.jp/photos/69d8596c65251ca8d6fddabb/master/w_2560%2Cc_limit/Backchannel-Is-Agentic-AI-Doomed-Business.jpg)

「AIエージェント元年」が毎年言われながら本格普及しない理由を、研究者と業界の両視点から分析した WIRED Japan の記事（2026年4月）。

## 懐疑論：数学的限界という主張

一部の研究者は LLM に**数学的に本質的な限界**があると主張する：
- LLM は本質的に「次のトークン予測」に過ぎず、真の計画立案・自律実行には不向き
- 原子力発電所のような critical system への適用は非現実的

## 楽観論：すでに機能している事例

業界側の反論：
- コーディング支援（GitHub Copilot、Claude Code 等）では実際に大きな成果
- Lean 等の数学検証ツールとの組み合わせで、幻覚（ハルシネーション）を構造的にトラップ可能
- ガードレールと人間の検証層の組み合わせで実用化できている

## 幻覚問題は解決不可能か

OpenAI の研究者も認める：**「精度が100%に達することは決してない」**

しかしこれは「使えない」を意味しない：
- 幻覚は「バグ」ではなく確率的推論の「仕様」
- 問題は幻覚そのものでなく、それをどう**検知・制御**するか
- 人間の判断が必要な場面（医療・法律・インフラ）では常に human-in-the-loop が必要

## 「元年」より「年次改善」というモデル

単一の革命的転換点（"agent year one"）を待つのではなく：
- 毎年、特定領域で実用性が増していく**漸進的モデル**が現実
- 「AIエージェントに何ができるか」より「**どう社会・職場が変わるか**」が本質的な問い

## 実務的示唆

| 向いているタスク | 向いていないタスク |
|-----------------|-----------------|
| コード生成・レビュー | セキュリティクリティカルな自律判断 |
| ドキュメント処理・要約 | 完全無監視の財務取引 |
| スケジュール・ルーティン自動化 | 医療診断（単体） |
| データ収集・分析 | 法的判断（単体） |

# lib0xc

Microsoft のオープンソース C 言語安全プログラミングライブラリ。C 標準ライブラリに隣接した API セットを提供し、システムプログラミングの安全性を高める。

## 設計思想

> "C を完全に型・境界安全にすることはできないが、その主要な使用パターンを現在より大幅に安全にできる"

- `-Wall -Wextra -Werror` での完全コンパイル対応
- 標準ライブラリ関数のドロップイン代替
- 固定サイズデータ構造で動的アロケーションを回避
- clang の `-fbounds-safety` エクステンションに完全対応

## API コンポーネント

### 標準ライブラリ拡張（`0xc/std/`）

| API | 機能 |
|-----|------|
| 安全な型付きアロケーション | ヒープ使用を型安全に |
| 境界チェック付きコンテキストポインタ | バッファオーバーフロー防止 |
| インメモリ I/O ストリーム | アロケーションなしのフォーマット |
| 安全な整数変換 | 整数オーバーフロー防止 |
| 静的文字列関数 | 文字列操作の安全化 |
| 構造体リフレクション | メタプログラミング支援 |

### システムプログラミングユーティリティ（`0xc/sys/`）

| API | 機能 |
|-----|------|
| バウンデッドバッファ | バッファ操作の境界管理 |
| オブジェクト指向ロギング | 構造化ログ |
| ハッシュテーブル・キュー | 境界安全アノテーション付き |
| 統一リンカセット | ELF・Mach-O 対応 |
| ユニットテストフレームワーク | 自動ディスカバリ付き |

## 要件

- C11/GNU 拡張
- clang または gcc
- macOS・Linux 対応

## 背景・文脈

Microsoft は Rust への移行（[[wiki/rust-for-cpython]] 参照）と同時に、既存の C コードを安全にするアプローチも進めている。lib0xc はその一環で、完全な言語移行が困難な大規模 C プロジェクト向けの実用的なソリューション。

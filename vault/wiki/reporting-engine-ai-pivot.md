# 帳票エンジン会社がGUIデザイナーを凍結してAIに全振りした話

名古屋のソフトウェア会社デンキヤギが、4年以上かけて開発したGUI帳票デザイナーの開発を凍結し、生成AIへの全面移行を決断した事例。

## 企業・製品概要

- **会社**：デンキヤギ（名古屋の小規模IT企業）
- **製品**：帳票エンジン「yagisan-reports」（2021年〜）

## 転換の経緯

**2025年初頭**：Cline導入とClaude Code Maxプランの登場で状況が変化。

「プロンプトを数回入力する程度で実用レベルの帳票テンプレートが生成される」ことを確認。

**2025年8月**：展示会でのデモが好評。既存ユーザーへのヒアリングでも「AIが実用的なら帳票デザイナーは不要」との声が一致し、経営判断が確定。

## 核心的な洞察

> **「GUIデザイナーは差別化にならない」**

競合との差別化要因になりえた機能が、生成AIの登場により「汎用品」になった。4年間の開発投資を捨てる決断。

## 今後の戦略

- 「人間がプログラムを書かない」社会構造に向けた開発戦略へ転換
- Agent Skillsという標準規格を採用し、特定AIツール（Claude Code等）への依存を回避

## 示唆

- ソフトウェア製品の「差別化機能」がAIで消失するリスクは幅広いドメインに存在する
- 小規模チームでも戦略的ピボットの決断がAI時代の競争力を左右する
- 標準仕様（Agent Skills等）を利用してツールロックインを避ける重要性

# セキュリティ分析 AI エージェントのハーネスエンジニアリング

![ハーネスエンジニアリング](https://storage.googleapis.com/zenn-user-upload/c96faff1081e-20260313.png)

Ubie のエンジニアが実装したセキュリティ分析 AI エージェント「Warren」を題材に、ハーネスエンジニアリングの実践的知見をまとめた Zenn 記事（2026-04-14）。

## ハーネスエンジニアリングとは

モデル自体ではなく、**エージェント全体の振る舞いを最適化する**エンジニアリング実践。プロンプト設計だけでなく、アーキテクチャ・評価・可観測性・継続改善サイクルを含む。

→ より詳細な定義: [[wiki/claude-code-harness-patterns]]

## Warren の3つのコア原則

1. **アーキテクチャで不確実性を制御** — プロンプトだけに頼らない
2. **寛容な入力、厳格な出力** — 入力は柔軟に受け入れ、出力は厳密に制御
3. **コスト最適化** — 無制限リソース消費を防ぐ予算管理

## 実装の3視点

### 計測（Measurement）
- 完全実行トレースの記録
- 受け入れ基準付き構造化タスク出力（合否明確化）

### 実行設計（Execution Design）
フェーズベースのフロー:
```
Intent → Plan → Execute → Replan → Final
```

- **ツール分離:** 機能ごとのツール分割
- **クレデンシャル分離:** Factory パターンで認証情報を隔離
- **安全ガードレール:** 重大操作前のヒューマンインザループ承認

### 最適化（Optimization）
- タスク実行後の自動反省（Reflection）で知識を抽出・蓄積
- ドメイン固有ランブック（Runbook）の整備
- 並列タスク実行でスループット向上
- 予算ベースのコスト制御（セッション単位の上限設定）

## BigQuery 多層ガード（事例）

セキュリティ分析でのデータベースアクセスを多層防御:

1. **ドライラン:** 実行前にクエリコスト見積もり
2. **スキャン制限:** 一定 GB 以上のスキャンをブロック
3. **タイムアウト:** 長時間クエリの自動中断
4. **外部通信の単一チョークポイント:** エグレスを一か所に集約

## 知識管理システム

タスク実行から2種類の知識を自動抽出:

| 種類 | 内容 |
|---|---|
| Technique | 「どうすれば解決できるか」の手順知識 |
| Fact | 「このシステムはどう動くか」の事実知識 |

重複検出・キュレーションでメモリブロートを防止。

# 社内AIアシスタントの3か月試験運用から得た教訓（松尾研究所）

![社内AIアシスタント試験運用](https://static.zenn.studio/user-upload/187a33179ebd-20260428.png)

松尾研究所が2026年2〜4月の3か月間、Slack ベースの AI アシスタント「mbot」を3プロジェクトで試験運用した実録。PM 業務の負荷軽減を目的とし、定量的な成果と課題の両方を公開。

## システム構成

- **ベース：** nanobot（軽量エージェントフレームワーク）
- **対応モデル：** Claude Code / Codex CLI
- **機能：** Slack 統合、GitHub Apps 認証、スケジュール自動実行、Agent Skills 拡張

## 主な活用事例

| ユースケース | 内容 |
|------------|------|
| 週報自動作成 | Notion 情報から自律的にまとめ |
| 論文・ニュース配信 | プロジェクト特化パーソナライズ配信 |
| コードレビュー | マルチエージェントで複数 LLM 検証 |

## 定量的成果（アンケート 8名回答）

| 指標 | 結果 |
|------|------|
| コードレビュー・資料検索の利用率 | 85.7% |
| 業務効率化評価 7以上（10点満点） | 100% |
| 業務効率化評価 9以上 | 62.5% |
| 週 30 分以上の時短達成 | 75% |
| 全社導入を「強く推す」 | 50% |

## 課題

1. **セットアップ難度** — クラウド環境構築・各種 API 設定が複雑で、非エンジニアには敷居が高い
2. **コスト** — 1プロジェクトで月 1〜2 万円（LLM API 費用）
3. **セキュリティ** — 権限管理とデータ破壊リスクの両立が困難

## 洞察

- 「週 30 分短縮」という控えめな成果でも全社展開を推す人が多いのは、**繰り返しタスクからの解放感**が評価されている証拠
- マルチエージェント構成（複数 LLM でコードレビュー）は品質向上に効果的だが、セットアップコストも倍増する
- セキュリティと利便性のトレードオフは、AI アシスタント普及の最大の障壁となっている

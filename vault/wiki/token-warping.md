# Token Warping

マルチモーダル LLM（MLLM）が**異なる視点からシーンを推論できるようにする推論時テクニック**。ピクセル空間ではなく**意味トークン空間**でワーピング（視点変換）を行うことで、学習・ファインチューニング・ピクセル合成を一切行わずにどの ViT ベース MLLM にも適用可能。

## 解決する問題

マルチビュー空間推論（あるカメラ視点から見た画像をもとに別の視点を理解する）は、従来のピクセルレベルのアプローチで行うと:

- 深度推定の小さな誤差に敏感（ピクセルレベルの幾何歪みが発生）
- 視点変換のためにリアルな画像を合成・再入力する必要がある

Token Warping はピクセルの代わりに **画像トークン**（ViT の patch embedding）を変換するため、これらの問題を回避する。

## 仕組み

### バックワードワーピング（推奨手法）

1. ターゲットビュー（推論したい視点）にグリッドを定義する
2. 深度マップからライトウェイトな 3D プロキシメッシュを構築する
3. レイキャスティングでターゲットの各グリッド点に対応するソース視点のトークン座標を計算する
4. トークンを最近傍で取得する（Nearest Fetching）

### トークン取得方式の比較

| 方式 | 精度 | 計算コスト |
|------|------|----------|
| **Nearest Fetching**（最近傍） | 高い | 最小（推奨） |
| Adaptive Fetching | わずかに高い | 高い（差は無視できる） |

最近傍取得は複雑な適応的手法とほぼ同等の性能を持ちながら、オーバーヘッドが最小。

## 認知科学的根拠

人間は視点変換を行う際、ピクセルではなく「パーツレベルの構造表現」を基盤として使用することが認知科学で示されている。Token Warping はこれを模倣しており、画像トークンが自然にパーツレベルの意味表現として機能する。

## パフォーマンス

### ViewBench ベンチマーク（5〜15% オーバーラップ）

| 手法 | Text 精度 | Shape 精度 |
|-----|---------|----------|
| ピクセルレベルワーピング | 低い | 低い |
| SpatialReasoner（ファインチューニング済み） | 中程度 | 中程度 |
| **Backward-Nearest（Token Warping）** | **57.81%** | **68.39%** |

フォワードワーピング（57.81% → 43.24%）と比較して +14.6 ポイント改善。

### 位置ノイズ耐性

画像トークンは 0〜20 ピクセルの位置摂動にわたって安定した精度を維持（ピクセルベースは急激に精度が落ちる）。

## 主な利点

- **学習不要**: 既存の ViT ベース MLLM にそのまま適用可能
- **推論オーバーヘッド最小**: Nearest Fetching は計算コストがほぼゼロ
- **深度エラー耐性**: 意味表現がピクセルより深度エラーに対して頑健
- **幻覚を起こさない**: GenWarp のような生成アプローチと異なり実在しない内容を生成しない

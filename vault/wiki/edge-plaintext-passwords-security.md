# Microsoft Edge：起動時に全パスワードを平文 RAM 展開（意図的設計）

ノルウェーのセキュリティ研究者が2026年4月に発見・公表した Microsoft Edge のセキュリティ上の懸念。Microsoft はこれを「意図的な設計」と確認。

## 問題の概要

Edge は起動時に保存されているすべてのパスワードを平文（暗号化なし）でメモリに展開する。Chrome や Brave は必要なときにのみ個別に復号するのとは対照的。

## ブラウザ比較

| 側面 | Edge | Chrome / Brave |
|------|------|----------------|
| パスワード復号タイミング | **起動時に全件ロード** | 必要時のみ個別復号 |
| 平文が存在する期間 | セッション全体（ブラウザを閉じるまで） | 最小限のウィンドウ |
| Microsoft の見解 | 意図的な設計 | — |

研究者は PoC（EdgeSavedPasswordsDumperPoC）を GitHub に公開し、2026年4月の BigBiteOfTech カンファレンスで実証。

## 実際のリスク評価

**前提条件**：悪用には管理者レベルのシステムアクセスが必要（Web ブラウジング単体では不可）

**特に危険なシナリオ**：
- 共有環境・ターミナルサーバー・リモートデスクトップ
- 企業内の内部サービス・クラウドプラットフォーム認証情報が保存されている場合
- 管理者権限を持つ内部犯またはマルウェアによる一括窃取

## 推奨対策

1. 機密性の高いクレデンシャルは専用パスワードマネージャー（1Password、Bitwarden 等）へ移行
2. 重要アカウントには MFA を必ず有効化
3. 共有デバイス・ターミナルサーバーでの Edge 使用を見直し
4. 日常業務での管理者権限使用を制限
5. Edge バックグラウンドプロセスの永続性設定を確認

## 補足

Microsoft が「意図的な設計」と述べている点は注目に値する——パフォーマンスと利便性を優先した設計選択が、セキュリティのトレードオフになっている。企業のセキュリティチームは Edge を標準ブラウザとして採用する際、この点を考慮すべき。

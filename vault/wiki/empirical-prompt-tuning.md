# 経験的プロンプトチューニング：AI による自動再現性向上

mizchi による手法（2026年4月）。別の AI エージェントにプロンプトを評価させるループを回すことで、作者バイアスを排除しながら再現性を自動向上させる。

## 問題：自己評価の構造的欠陥

プロンプトを書いた人が自分でレビューすると、**無意識に空白を補完**してしまう。これは「技量不足」ではなく認知の構造的問題。外部の判断者が必要。

## 手法の核心

Claude Code の `Task` ツールを使い、**独立した別の AI インスタンス**にプロンプトを評価させる。

### プロセス

```
1. テストシナリオ設定（中央値ケース + エッジケース）
2. 評価 AI に送信 → 不明点・補完した仮定・要件充足率をレポート
3. 1サイクル1修正（変更を限定して因果を追跡）
4. 新鮮な AI インスタンスで評価（バイアス蓄積防止）
5. ホールドアウトシナリオでオーバーフィット検出
```

### 停止条件

2サイクル連続で：
- スコア改善 < 3ポイント
- 実行時間の分散 < 15%

## 計測指標

| 指標 | 意味 |
|------|------|
| `tool_uses` 数 | プロンプトの明確さの代理指標 |
| 実行時間 | 安定性の測定 |
| チェックリスト達成率 | 要件充足度 |

## 実績

著者自身がこの記事をこの手法で7回イテレーション：再現性スコアが **6/20 → 16/20** に向上。

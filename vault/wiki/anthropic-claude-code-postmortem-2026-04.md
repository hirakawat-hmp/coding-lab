# Anthropic Claude Code 品質問題ポストモーテム（2026年4月）

2026年3月〜4月にかけて報告された Claude Code の応答品質低下問題に関する Anthropic の公式ポストモーテム。3つの独立した問題が特定・修正された（全て 2026-04-20 までに解決）。

## 問題 1：推論努力レベルのデフォルト変更

**期間**: 2026-03-04（変更）→ 2026-04-07（差し戻し）

UI が凍結するほどのレイテンシ問題を解消するため、Claude のデフォルト推論努力レベルを **「高」→「中」** に変更した。ユーザーからの反発（「回答が浅い」）を受け、Opus 4.7 は「xhigh」、他モデルは「高」に戻した。

## 問題 2：キャッシュ最適化バグ

**期間**: 2026-03-26（導入）→ 2026-04-10（修正）

アイドルセッション管理機能のバグ。推論履歴を「1回だけ」削除すべきところ、**セッション中ずっと削除し続けた**。ユーザーには「もの忘れっぽい」「繰り返しが多い」と体験される。

## 問題 3：システムプロンプトの冗長性削減指示

**期間**: 2026-04-16（追加）→ 2026-04-20（削除）

「ツール呼び出し間のテキストは25語以下に」という簡潔化指示をシステムプロンプトに追加。評価で **3% の性能低下**を確認し即座に削除した。

## 再発防止策

- 全加入者の利用制限をリセット
- Code Review ツールの改善と内部展開
- **システムプロンプト変更時の広範な評価実施**を義務化
- 段階的ロールアウトと「浸潤期間」の導入

## 教訓

- 運用上の最適化（レイテンシ改善）が品質に影響する可能性
- キャッシュ・セッション管理のバグはユーザー体験に直接現れる
- システムプロンプトの変更は小さくても評価結果に影響する

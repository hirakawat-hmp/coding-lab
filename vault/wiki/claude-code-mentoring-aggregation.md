# Claude Codeで育成情報を集約した1on1自動化の実録

ソニックガーデン社の「親方」が5名の若手エンジニア育成情報をClaude Codeで一元管理し、1on1下書きや振り返り資料の作成を効率化した事例（2026年5月報告）。

## 課題

育成に関連する情報が複数のツールに分散していた：
- 観察メモ（メモアプリ）
- ふりかえり記録（別ドキュメント）
- 日報（社内SNS）
- 育成会議メモ（スプレッドシート）

→ 一貫した判断軸の維持が困難、管理業務に時間がかかる

## 実装アーキテクチャ

```
観察メモ
1on1記録          → CLAUDE.md（業務マニュアル・文体ルール定義）
日報                         ↓
知識ベース      Claude Codeが集約・整理・下書き生成
```

## 実際の使い方

| コマンド例 | 動作 |
|------------|------|
| 「Aさんの様子を記録して」 | 日常メモを自動追記 |
| 「Google Docsから取り込んで」 | 複数ツールの情報を統合 |
| 「振り返りの下書きを作って」 | 評価軸に沿った下書きを自動生成 |

## 重要な原則

> 「任せているのは**思考の下ごしらえ**。AIの出力は必ず事実確認し、観察そのものは親方が担当」

AIは情報の整理・下書き生成を担い、最終判断と観察は人間が担当するという明確な役割分担。

## 成功の鍵

**暗黙知の言語化**: CLAUDE.mdに育成の哲学・評価軸・文体ルールを明文化する `knowledge/` ディレクトリが核心。ここを整備する前はAIの出力がブレた。

## 汎化できるパターン

- 人事評価・メンタリング記録の集約
- プロジェクト関係者ノートのまとめ
- 複数ツールにまたがる議事録・メモの横断検索

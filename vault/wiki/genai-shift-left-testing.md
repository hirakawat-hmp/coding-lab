# 生成AIによるシフトレフト — コードを書く前にバグを潰す

生成AIを使い、テストやバグ検出を開発プロセスの**最上流（仕様策定段階）**に移動させるアプローチ。従来は理想論だったシフトレフトが、AIツールにより現実的になった。

## 仕様レベル TDD

著者が提唱するコアアプローチ：TDD（テスト駆動開発）の思考を仕様策定段階に適用する。

```
[従来のTDD]
コード → テスト → 修正 → コード...

[仕様レベルTDD]
仕様ドラフト → テスト観点 → ギャップ分析 → 仕様修正 → 実装
```

**プロセス：**
1. AIが仕様ドラフトとテスト観点を**同時に**生成
2. テスト観点を仕様にぶつけてギャップを発見
3. 仕様を修正するループを数分〜数時間で完走
4. 実装前に仕様品質を大幅に向上させてから実装に入る

## 検出される欠陥パターン（18プロジェクト以上の実績）

- **エラーハンドリングの未定義**
- **境界値の仕様未定義**
- **権限チェックの考慮漏れ**
- **状態遷移の矛盾**

論理的な仕様バグをほぼ毎回発見できる。

## QAの役割変化

| 従来 | AI時代 |
|------|--------|
| バグを発見する | バグを生まない仕様を作る |
| 実装後の検証者 | 仕様の品質保証者 |
| ブラックボックステスト | テスト観点・分析力が核能力 |

「外部品質（ユーザー視点）の代弁者」としての立場が強化される。

## ツール

- コード生成：Claude Code、GitHub Copilot
- テスト自動化：Playwright Agents
- 仕様レビュー：LLM の論理推論能力

# AI エンジニアリングの進化：プロンプト → コンテキスト → ハーネス（2022-2026）

2022年から2026年にかけての AI エンジニアリングの3フェーズ進化を整理した概念。
現在は「ハーネスエンジニアリング」が中心になりつつある。

## 3フェーズの変遷

### Phase 1: プロンプトエンジニアリング（2022-2023）

- **焦点:** LLM との直接 Q&A インタラクション
- **技術:** Zero-shot・Few-shot・CoT・ペルソナ付与
- **時代の特徴:** 「指示の時代」—— モデルを呪文で動かす

### Phase 2: コンテキストエンジニアリング（2024-2025）

- **焦点:** 拡張コンテキストウィンドウを通じた情報管理
- **技術:** RAG・ベクトル DB・チャンキング・ハイブリッド検索・再ランキング
- **時代の特徴:** 「記憶の時代」—— 何をコンテキストに入れるかが勝負

### Phase 3: ハーネスエンジニアリング（2025-2026〜）

- **焦点:** AI エージェントを安全に動かす構造的フレームワーク設計
- **技術:** オーケストレーション・状態管理・ガードレール・サンドボックス・観測性
- **時代の特徴:** 「制御の時代」—— 従来コードで AI の行動範囲を画する

## ハーネスシステムの5本柱

1. **オーケストレーション** — 専門役割を持つマルチエージェントとグラフワークフロー
2. **状態・メモリ管理** — チェックポイント・長期設定・アイドル中の知識統合
3. **ガードレール・バリデーション** — 構造化出力強制・自己修正ループ・API ゲートウェイ
4. **ツール利用・サンドボックス化** — Function Calling・MCP・隔離実行環境
5. **オブザーバビリティ** — LLM トレーシングと思考プロセスの透明性

## ビッグテック実績

| 企業 | 実績 |
|------|------|
| Anthropic | 16エージェントが 2週間で 10万行 C コンパイラ。堅牢性の80% はエンジニアリングされたセーフガード |
| OpenAI | 5ヶ月で 100万行 SaaS（人間の書いたビジネスロジックなし） |
| Google | AI 駆動 CI/CD で自動自己修復コード |

## 役割の変化

```
従来: Engineer = コーダー（関数を書く）
現在: Engineer = Agentic Architect（自律システムを設計・監視）
```

エンジニアに求められるスキルセットが、実装から設計・制御・観測へとシフトしている。

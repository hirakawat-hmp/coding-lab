# Agentic UAT — AIエージェントによるユーザー受け入れテスト

AIエージェントがエンドユーザーを演じて業務フローを検証する手法。BizReach AI Product Studio が提唱。「テストは通るがユーザー視点では動かない」バグを、実装者視点のテストでは検出できないという課題への解答。

## E2Eテストとの根本的差異

| 観点 | E2Eテスト | Agentic UAT |
|------|-----------|------------|
| 知識 | URLや手順を知っている実装者が設計 | 仕様知識なし・UIから手順を発見 |
| 指示 | 手順を指定（クリック先・URL） | 業務ゴールのみを指定 |
| 検証内容 | 機能の動作確認 | UIの発見性・直感的操作性も含む |

```
# E2Eテスト的指示
1. /users/new にアクセス
2. 名前欄に入力
3. 保存ボタンをクリック

# Agentic UAT 的指示
新しいユーザーを登録してください。
登録後、ステータスを有効にしてください。
```

## エージェント構成

```
UATマネージャー（業務フロー定義を読み込む）
  ├── ペルソナA（営業担当者・新人）
  ├── ペルソナB（管理担当者・ベテラン）
  └── ペルソナC（別ロール）
```

### エージェントへの制約（「素人ユーザー」化）

- ブラウザUI操作のみ（API直接呼び出し禁止）
- ソースコード読取禁止
- 仕様書の参照禁止
- 詰まり方そのものを記録

## ペルソナ設計

「担当業務」 × 「業務習熟度」の2軸。同じ画面でも新担当者とベテランでは詰まるポイントが異なる。

## 問題分類

| 分類 | 定義 |
|------|------|
| **Critical** | 業務完遂不可 |
| **Major** | 完遂可だが大きな障壁 |
| **Minor** | 改善余地程度 |

## 適用場面

- AIが実装した機能のUAT（仕様充足確認）
- 機能完成時・マイルストーン前の節目での実行
- PR毎の人間による手動UATコスト削減

## 限界

- **視覚的課題**：レイアウトの微妙なおかしさや美的評価は不得意
- **ペルソナの不完全性**：属性が実際の行動パターンを完全に再現できない
- **実行時間**：数十分に及ぶため毎PR実行は現実的でない
- **スコープの限定**：「使いにくさの直感」や業界固有慣習は検出不可

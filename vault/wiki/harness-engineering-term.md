# ハーネスエンジニアリングという用語の解体

2026年4月に SNS で急増した「ハーネスエンジニアリング」という用語の批判的検討。既存実践の再ラベリングか、有意義な新概念か。

## ハーネスの語源

Cambridge Dictionary: 「ベルトとストラップで構成された、人や動物・物体を**制御または固定する**装置」

AI エージェントの文脈では「モデルをコントロール下に置く仕組み全体」。

## LangChain による定義

```
Agent = Model + Harness
```

**ハーネス**（モデル以外のすべて）:
- システムプロンプト
- ツール / スキル / MCP サーバー
- インフラ・デプロイ環境
- オーケストレーションロジック
- フック・ライフサイクル管理

## 「新語か既存実践か」論争

### 新語と見る立場

「ハーネスエンジニアリング」という概念の明示化により、モデル側の改善（プロンプト → RAG）からインフラ側の整備へと議論の軸が移ることを促す。

### 既存実践の再ラベルと見る立場（本記事の主張）

Claude Code のスキル・フック・フィードバックループを実装している開発者は、すでにハーネスエンジニアリングを実践している。バズワードは**既存実践への事後命名**に過ぎない。

## 価値がある文脈

コーディング領域を超えた場面で意義を持つ:

1. **特定ビジネスドメイン向け AI エージェントの開発** — 業務プロセスの知識をハーネスに込める
2. **エンドユーザーによるカスタマイズ** — ノーコード・ローコードでハーネスを調整させる製品設計

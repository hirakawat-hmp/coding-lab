# バイブコーディングとアジェンティック・エンジニアリングの境界消滅

Simon Willison（datasette 作者）が2026年5月に指摘した重要な観察：かつて明確だった「バイブコーディング」と「アジェンティック・エンジニアリング」の区別が、実践上では曖昧になりつつある。

## 2つのアプローチの定義

| アプローチ | 定義 |
|------------|------|
| バイブコーディング | コードを理解せずに書く（内容を把握していない） |
| アジェンティック・エンジニアリング | AI を活用しながら品質標準を維持するプロ開発者のアプローチ |

## なぜ境界が消えているか

### 責任の問題
Willison 自身、本番システムに対して AI エージェントが生成したコードを全行レビューするのをやめていると告白。Claude Code は安定した成果物を生み出すが、「レビューしていないコードを本番で使うのは責任ある行為か？」という倫理的な緊張が生じる。

解決策として：AI エージェントを大組織の信頼できるチームのように扱い、問題が起きるまで成果物を信頼する。

### 評価の困難化
リポジトリが慎重に作られたものか、AI で素早く生成されたものか、見た目では区別がつかない。Willison は今や、磨き込まれたドキュメントより**実際の使用実績**を信頼度の指標とする。

### 構造的な混乱
生産性が200行/日から2,000行/日に向上したことで、上流の設計ボトルネックが露わになった。ソフトウェア開発ライフサイクル全体がより遅いコード生産速度を前提として設計されており、ワークフロー全体の再設計が必要になっている。

## 開発者のキャリアへの影響

Willison はプロ開発者の将来に楽観的。AI ツールは「既存の専門知識を効果的に使用するために専門知識を必要とする増幅器」であり、経験のある開発者ほど恩恵が大きい。

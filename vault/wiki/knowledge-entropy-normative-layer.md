# 知識エントロピーと規範レイヤー：組織出力品質の本質

「人を増やすか減らすか」という二元論を超え、組織の出力品質を決める本質的要因を分析した概念フレームワーク。AI時代の組織設計に適用できる。

## 知識エントロピーとは

メンバー間の知識分布のばらつき度合い。エントロピーが高いほど、意思決定のたびに「すり合わせが必要な情報量」が増える。

**エントロピー増加の連鎖**：
```
知識エントロピー増加
  → 文脈共有時間の増加
  → 意思決定速度の低下
  → リード層への依存強化
  → 個人の認知負荷増加
  → さらなる品質低下
```

## AIエージェント導入だけでは解決しない3問題

| 問題 | メカニズム |
|------|-----------|
| 一貫性の喪失 | 局所最適な判断の累積がUI・データモデルのばらつきを招く |
| ナレッジの劣化 | AI生成知識の矛盾と陳腐化が加速度的に進行 |
| 燃え尽き | 自律的人材への判断集中が認知負荷を増加させ育成を圧迫 |

人を増やしても、エントロピーを下げる仕組みがないと問題は解決しない。

## 規範レイヤーの設計

解決策は「自動検出可能な少数の条文」から構成される規範レイヤー：

- 型検査やCI/Linterで自動検出される
- 「組織が何を許容し、何を禁じるか」を明確化
- **過度な詳細化は避け、最重要の5〜10条に絞り込む**

Accelerate研究が示す「生成的な組織文化」を維持しながら、最小限の規範を整備することが重要。

## CLAUDE.md との類比

この「規範レイヤー」は、vault における [[wiki/agents-skill-design-md-roles]] の `AGENTS.md/CLAUDE.md` 層と同じ設計思想。：
- エージェント（人間・AI）が守るべき「少数の重要ルール」を定義
- 詳細すぎると機能しない（官僚化）
- 自動検証できる形式にする

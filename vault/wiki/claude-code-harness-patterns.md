# Claude Code ハーネスパターン10選 — ソースコード流出から学ぶ制御層設計

## 背景: ソースコード流出事件

2026年3月31日、Claude Code v2.1.88 の npm パッケージに
59.8MB のソースマップが誤同梱され、
**512,000行・1,906ファイル**の TypeScript コードが復元可能になった。

原因: `.npmignore` での `.map` 除外漏れ × Bun ランタイムの既知バグ

流出後も Anthropic の競争優位が揺らがない理由:
**コードの背後にある組織知と数年の運用知見は再現不可能**

## 核心思想

> **「モデルは交換可能、ハーネスは経験の結晶」**

AI エージェントの品質は推論モデルよりも**制御層（ハーネス）の設計**に依存する。

## 10のハーネスパターン

### 1. ハーネスパラダイム — モデルと制御の分離

ハーネスが解決する5課題:
- アクション空間の制約
- 状態管理
- パーミッション
- 障害処理
- リソース最適化

### 2. ツールコントラクト — Zod による統一インターフェース

すべてのツールを Zod スキーマで定義し、
ID / ロジック / バリデーション / パーミッション / 表示を標準化。
→ 保守性と再利用性が大幅に向上。

### 3. クエリエンジン — 会話ライフサイクルの中央管理

エラー時の段階的回復戦略（優先順位順）:
```
コンテキスト折りたたみ
  → リアクティブコンパクション
    → トークンエスカレーション
      → モデルエスカレーション
        → 安全停止
```

### 4. パーミッションシステム — 5層パイプライン

```
Allow/Deny/Ask リスト
  → ツール固有チェック
    → 動的分類器
      → ユーザー承認
        → 監査ログ
```
Bash 実行だけで 23 のセキュリティチェックを実施。

### 5. コンテキストエントロピー管理 — 3層メモリ構造

| 層 | 内容 | 特記 |
|---|---|---|
| 短期 | 現在の会話 | 高精度 |
| セッション永続化 | インデックス + トピック分散 | 中程度の精度 |
| 長期メモリ | 「権威的事実」ではなく「ヒント」扱い | ハルシネーション永続化防止 |

### 6. プロンプトキャッシュ最適化 — 静的/動的の分離

- **静的命令**（セッション間で不変）→ CLAUDE.md 上部に配置
- **動的コンテキスト**（毎セッション変化）→ 末尾に追加

→ API キャッシュ機構を最大活用してコスト削減。

### 7. 反蒸留メカニズム — 競合の学習防止

デコイツール定義注入 + アシスタントテキスト要約により、
API 傍受による競合のモデル学習コストを意図的に高める。

### 8. フラストレーション検出 — 推論外感情分析

正規表現で低コストな感情分析を実行。
モデルに渡す前にユーザー状態を推論する状態管理パターン。

### 9. アンダーカバーモード — 環境適応パターン

実行環境に応じて振る舞いを動的に切り替え。
「AI 関与の痕跡を残さない」環境適応を実現。

### 10. ネイティブクライアント認証 — APIのDRM

Bun の Zig 実装ネイティブ層でハッシュ値を計算・署名。
JavaScript レベルでのリバースエンジニアリングを困難化する「API の DRM」機構。

## 未出荷機能

**KAIROS**: ユーザー指示なしの自発的バックグラウンドデーモン
```
観察ログ → パターン検出 → トリガー判定 → 夜間メモリ蒸留
```

**ULTRAPLAN**: 計画フェーズをクラウドにオフロードし、30分の「熟考」後にローカル実行する計画/実行分離パターン。

## 個人開発者への応用優先度

1. **パターン5・6**（メモリ管理とキャッシュ最適化）→ 設定の並べ替えで即効果
2. **パターン1・4**（ハーネスパラダイム・パーミッション）→ 安全性と再現性向上

実践的な設定例:
- CLAUDE.md 上部に低頻度変更項目（基本ルール）を配置
- `.claude/commands/` でツールに入出力スキーマを統一記述
- `settings.json` で Deny/Allow リストを定義

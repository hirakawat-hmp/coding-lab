# Daily Knowledge Digest — 2026-04-30

## 要約

Claude Code の重大な課金バグ（HERMES.md）が最大の話題となった一方、Linux カーネルの LPE 脆弱性・Ramp Sheets AI のデータ漏洩と AI ツールのセキュリティインシデントが続いた。技術面では Mistral Medium 3.5（Open weights SWE-Bench 77.6%）・Zed 1.0 GA・JavaScript Temporal API 標準化が注目。GitHub の連続障害も6日間の危機として総括された。

## トピック

### 1. Claude Code HERMES.md コミットメッセージで Max プランが追加課金される

コミットメッセージに `HERMES.md`（大文字・case-sensitive）が含まれると、Claude Code が Max プランの利用枠ではなく「extra usage（追加課金）」にルーティングされる。報告者は $200.98+ が意図せず消費された（プラン枠は86%以上残存）。HN で 944 ポイントを獲得し、最注目バグとなった。

対処法: コミットメッセージに `HERMES.md` を使わない。小文字 `hermes.md` は影響を受けない。

**関連ページ:** [[wiki/claude-code-hermes-billing-bug]]
**ソース:** [GitHub Issue #53262](https://github.com/anthropics/claude-code/issues/53262)

### 2. CVE-2026-31431 "Copy Fail" — Linux カーネル LPE 脆弱性

2017年以降のすべての主要 Linux ディストリビューションに影響するローカル特権昇格脆弱性。`authencesn` のロジックバグが `AF_ALG` + `splice()` を経由して setuid バイナリのページキャッシュに4バイト書き込みを可能にする。732バイトの Python スクリプトでエクスプロイト可能。約9年間、誰にも気づかれなかった。

**関連ページ:** [[wiki/cve-2026-31431-linux-kernel]]
**ソース:** [copy.fail – CVE-2026-31431](https://copy.fail/)

### 7. コンパクト・ボンネペア・トーラス — 150年の幾何学定説を覆す発見

![コンパクト・ボンネペア・トーラス](https://media.wired.jp/photos/69f1a749bd05cfe5348ee47c/master/w_2560%2Cc_limit/csm_20260312_Loesung_Bonnet_Problem_AE_-10_01_7fcd487d88.jpeg)

ミュンヘン工科大学等の研究チームが、同一の計量と平均曲率を持つのに3D空間で重ね合わせられない2つのトーラス曲面を初めて明示的に構築。19世紀数学者 Bonnet の「曲率が形状を一意に決定する」という仮定を否定した。わずか 35点（5×7グリッド）の離散モデルから得た洞察が連続解への橋渡しとなった点が印象的。

### 9. FastCGI はリバースプロキシで HTTP より優れている（30年間見過ごされた設計）

HTTP/1.1 のメッセージフレーミング欠如がリクエストスマグリング攻撃の根本原因であることを解説したエッセイ（HN 224 ポイント）。30年前のプロトコル FastCGI は明確なメッセージ境界と信頼データの構造的分離を実装しており、nginx・Apache・Caddy・HAProxy がサポートしている。WebSocket 非対応が制限だが、Web バックエンドとのリバースプロキシ接続では構造的に優れている。

**関連ページ:** [[wiki/fastcgi-reverse-proxy-advantage]]
**ソース:** [agwa.name](https://www.agwa.name/blog/post/fastcgi_is_the_better_protocol_for_reverse_proxies)

### 11. Lakebase × LangGraph × Qwen3 — セッションを跨ぐ AI エージェントメモリ設計

### 12. JavaScript Temporal API — 9年かけて Date を置き換える

![JavaScript Temporal API](https://i.gzn.jp/img/2026/04/29/fix-time-in-javascript/00_m.jpg)

1995年に10日間で実装された JavaScript `Date` オブジェクトの置き換えとなる Temporal API が TC39 Stage 3 に到達。2017年から約9年の開発期間を経た。不変性・複数型・タイムゾーン/カレンダーのファーストクラスサポートが特徴。Test262 に 4,500件のテストケースを持ち、他の組み込み機能を大幅に超えるテストカバレッジ。

**関連ページ:** [[wiki/javascript-temporal-api]]
**ソース:** [Gigazine](https://gigazine.net/news/20260429-fix-time-in-javascript/)

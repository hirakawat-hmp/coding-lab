### 1. AIが数学研究を変え始めた

Quanta Magazine報道（2026-04-13）。テレンス・タオ（UCLA）をはじめ世界トップの数学者が実際の研究にAIを使い始め、「今後数学は今までとは全く異なるものになる」と言及。

DeepMindのAlphaEvolveは4×4行列乗算で57年ぶりの記録を更新（50スカラー演算 → 48回）。5人の数学者がAIを使って順列群における「超立方体構造」を発見し、「50年間誰も気づかなかった」と述べた。2026年2月のFirst Proofチャレンジでは、研究レベルの数学問題10問のうち5問以上をAIが1週間で解いた。

ただし懸念もある。AI生成の誤情報が学術誌を汚染するリスク、教育への悪影響、アクセス格差の拡大が指摘されている。

**関連ページ:** [[wiki/ai-math-revolution]]
**ソース:** [The AI Revolution in Math Has Arrived](https://www.quantamagazine.org/the-ai-revolution-in-math-has-arrived-20260413/)

### 2. N-Day-Bench：LLMがリアルな脆弱性を発見できるか

Winfunc Researchが運用するベンチマーク。知識カットオフ後に公開された実世界の脆弱性（N-Day）をLLMが発見できるかを月次評価する。

2026年4月13日の最新結果ではGPT-5.4が83.93点でトップ、Claude-Opus-4.6が3位（79.95点）。47件のアドバイザリを対象とし、全テストのトレースを公開している。

**関連ページ:** [[wiki/n-day-bench]]
**ソース:** [N-Day-Bench](https://ndaybench.winfunc.com)

### 3. WordPressプラグイン30本に一斉バックドア埋め込み

攻撃者がFlippaでプラグインポートフォリオを買収し、2025年8月に悪意のあるコードを仕込んで8ヶ月潜伏、2026年4月に発動した。

技術的に注目すべき点は2つ：`@unserialize()`を悪用した任意関数実行と、**C2ドメインをEthereumスマートコントラクトで管理**することで従来のドメイン無効化対策を無効化したこと。WordPress.orgは強制更新で対応したが、所有権変更時の審査メカニズムの欠如という根本問題は未解決。

**関連ページ:** [[wiki/wordpress-plugin-supply-chain-attack]]
**ソース:** [Someone bought 30 WordPress plugins and planted a backdoor](https://anchor.host/someone-bought-30-wordpress-plugins-and-planted-a-backdoor-in-all-of-them/)

### 4. Lean形式検証の「証明済みコードにバグが見つかった」

形式証明支援系Leanで検証済みのzlib実装に、ClaudeによるAIファジング（1億500万回実行）でバグが2件発見された。

1件目はLeanランタイム自体（C++）のヒープバッファオーバーフロー、2件目はアーカイブパーサーのDoS。形式検証は「検証した命題の範囲内では絶対正しい」が、「何を検証するか」は人間が決め、ランタイムへの信頼も前提とする。この事例は形式検証とテストが補完関係にあることを改めて示した。

**関連ページ:** [[wiki/lean-formal-verification-limits]]
**ソース:** [Lean proved this program correct; then I found a bug](https://kirancodes.me/posts/log-who-watches-the-watchers.html)

### 5. AWS App Runner、4月30日で新規受付停止 — 大規模サービス終了計画

AWSが複数サービスの終了計画を発表。App Runnerは2026年4月30日に新規受付停止（代替：Amazon ECS Express Mode）。Amazon RDS Custom for Oracleは2027年3月31日終了。

さらに8サービスがメンテナンスモード移行（Amazon Comprehend、Rekognition一部機能、AWS Audit Manager等）、3サービスがサンセット段階（Amazon WorkMail等）。AWSの製品ラインナップのスリム化が加速している。

### 6. AWS Frontier Agents GA：SREの仕事が変わる、消えるわけではない

AWS DevOps AgentとSecurity AgentがGA（2026年3月31日）。DevOps Agentは複数監視ツールを横断してインシデントの根本原因を特定し、MTTR最大75%削減を実証。Security Agentは継続的ペネトレーションテスト（$50/タスク時間）、設計レビュー、PRセキュリティレビューを提供。

SREの「仕事が消える」わけではなく「役割が変わる」。深夜の調査時間が減る一方、「エージェント出力の正確性判断」という新たな認知負荷が生まれる。余った時間を人員削減に充てるのは危険で、プロアクティブな改善に回すべき。

**関連ページ:** [[wiki/aws-frontier-agents]]
**ソース:** [AWS Frontier AgentsでSREの仕事が変わる](https://zenn.dev/genda_jp/articles/349257b6d83fd2)

### 8. Ingress NGINX のリタイアメントに伴う Gateway API + Envoy Gateway への移行

ingress-nginxが2026年3月にretirement。みてねチームのEnvoy Gateway移行実践レポート。

移行の起点は「使用中のannotationを全て列挙する」こと。Gateway APIを選んだ理由は中長期の拡張性と、責務（ルーティング・認証・ポリシー）を明示的に分離できる点。段階的切り替えには「ALBリスナールールで新旧共存 + ?debug=trueで検証」という手法を採用した。

**関連ページ:** [[wiki/ingress-nginx-to-gateway-api]]
**ソース:** [Ingress NGINXのretirementに伴うGateway API + Envoy Gatewayへの移行](https://zenn.dev/mitene/articles/202604-ingress-nginx-to-envoy-gateway)

### 9. Cloudflare統合CLI「cf」技術プレビュー — ローカル開発とエージェント対応

Cloudflareが全製品を統合する新CLI`cf`を技術プレビュー公開。約3,000のHTTP API操作に対応し、`npx cf`で試用可能。

新機能「Local Explorer」により、KV、R2、D1、Durable ObjectsなどをリモートAPIと同じインターフェースでローカルから操作できる。AIエージェント向けに`--json`・`--force`フラグを全コマンドで統一し、ローカル/リモートの区別を明確化している。TypeScriptベースの独自スキーマシステムからCLI・Bindings・設定ファイルを一括自動生成。

**関連ページ:** [[wiki/cloudflare-cf-cli]]
**ソース:** [Building a CLI for all of Cloudflare](https://blog.cloudflare.com/cf-cli-local-explorer/)

### 11. SnapState：AIエージェントの永続状態管理

JavaScript/Python SDK、MCP対応。無料枠：月10,000書き込み・1GB・5,000再開。

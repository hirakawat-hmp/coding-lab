### 1. AIが数学研究を変え始めた

Quanta Magazine報道（2026-04-13）。テレンス・タオ（UCLA）をはじめ世界トップの数学者が実際の研究にAIを使い始め、「今後数学は今までとは全く異なるものになる」と言及。

DeepMindのAlphaEvolveは4×4行列乗算で57年ぶりの記録を更新（50スカラー演算 → 48回）。5人の数学者がAIを使って順列群における「超立方体構造」を発見し、「50年間誰も気づかなかった」と述べた。2026年2月のFirst Proofチャレンジでは、研究レベルの数学問題10問のうち5問以上をAIが1週間で解いた。

ただし懸念もある。AI生成の誤情報が学術誌を汚染するリスク、教育への悪影響、アクセス格差の拡大が指摘されている。

### 3. WordPressプラグイン30本に一斉バックドア埋め込み

攻撃者がFlippaでプラグインポートフォリオを買収し、2025年8月に悪意のあるコードを仕込んで8ヶ月潜伏、2026年4月に発動した。

技術的に注目すべき点は2つ：`@unserialize()`を悪用した任意関数実行と、**C2ドメインをEthereumスマートコントラクトで管理**することで従来のドメイン無効化対策を無効化したこと。WordPress.orgは強制更新で対応したが、所有権変更時の審査メカニズムの欠如という根本問題は未解決。

### 4. Lean形式検証の「証明済みコードにバグが見つかった」

形式証明支援系Leanで検証済みのzlib実装に、ClaudeによるAIファジング（1億500万回実行）でバグが2件発見された。

### 6. AWS Frontier Agents GA：SREの仕事が変わる、消えるわけではない

AWS DevOps AgentとSecurity AgentがGA（2026年3月31日）。DevOps Agentは複数監視ツールを横断してインシデントの根本原因を特定し、MTTR最大75%削減を実証。Security Agentは継続的ペネトレーションテスト（$50/タスク時間）、設計レビュー、PRセキュリティレビューを提供。

SREの「仕事が消える」わけではなく「役割が変わる」。深夜の調査時間が減る一方、「エージェント出力の正確性判断」という新たな認知負荷が生まれる。余った時間を人員削減に充てるのは危険で、プロアクティブな改善に回すべき。

**関連ページ:** [[wiki/aws-frontier-agents]]
**ソース:** [AWS Frontier AgentsでSREの仕事が変わる](https://zenn.dev/genda_jp/articles/349257b6d83fd2)

### 8. Ingress NGINX のリタイアメントに伴う Gateway API + Envoy Gateway への移行

### 14. axios CVSS 10.0 脆弱性 — 評価の信頼性を巡る論争

axios（npm 週間ダウンロード数 数億）に CVSS 10.0 の最高評価が割り当てられた脆弱性（GHSA-fvcv-3m26-pcqx / CVE-2026-40175）が公開された。CRLF インジェクション + プロトタイプ汚染の組み合わせによる HTTP リクエストスマグリングで、最悪ケースでは AWS IMDSv2 認証情報の窃取によるクラウドアカウント乗っ取りが可能とされる。

しかし複数のエンジニアが Node.js での PoC 再現に失敗。Node.js の `http` モジュールが CRLF を `assertValidHeaderValue` で拒否するため、実際の危険性は**ブラウザや React Native 環境に限定される可能性が高い**。CVSS 10.0 評価の適切性に疑問符が付いた事例。対処：`npm install axios@^1.15.0`。

**関連ページ:** [[wiki/axios-cve-2026]]
**ソース:** [axiosで発見された重大な脆弱性](https://zenn.dev/ashunar0/articles/108d06e3f99efb) / [GHSA-fvcv-3m26-pcqx がなんか変](https://zenn.dev/masakura/articles/bc88704f8af091)

### 16. Bun v1.3.12 Headless Browser Automation — AIエージェントがブラウザを直接操作

Bun v1.3.12 に `bunwv` コマンドが追加され、コマンドラインから直接ヘッドレスブラウザを操作できるようになった。`bunwv start` でブラウザ起動、`bunwv navigate URL` でページ遷移、`bunwv screenshot` でスクリーンショット、`bunwv click-text` でテキストリンクをクリック。

macOS では OS レベルの操作のためネイティブとの区別が不可能。Claude Code などの AI エージェントが Playwright/Puppeteer 不要で Web ブラウザを操作できるようになり、テスト自動化・スクレイピング・フォーム操作が大幅に簡略化される。

**関連ページ:** [[wiki/bun-headless-browser]]
**ソース:** [Bun、Headless Browser Automation 搭載](https://www.publickey1.jp/blog/26/bunheadless_brwoser_automation.html)

### 17. SKILL.md パターン — Claude Codeの折りたたみ式マニュアル

CLAUDE.md がプロジェクト全体のルールを常時ロードするのに対し、SKILL.md は「呼び出された時だけロードされる折りたたみ式マニュアル」として機能する。frontmatter（`name` + `description`）のみが初期ロードされ、本体はスキル実行時に展開される。

実装例: 記事執筆スキル（ユーザー承認ポイントを明示したフロー）、コードレビュースキル（5軸評価テーブル出力）、デプロイスキル（本番前の明示的承認要求）。繰り返しパターンを1つずつスキル化してチームで共有・バージョン管理することで、AI 協業の品質をチーム全体で均質化できる。

**関連ページ:** [[wiki/skill-md-pattern]]
**ソース:** [SKILL.md で Claude Code ワークフローを自動化する](https://qiita.com/nogataka/items/c59defafd0dfb88c4a90)

### 18. Rust for CPython — Python 3.16 への Rust 統合が始動

CPython に Rust を段階的に統合するイニシアティブが正式な PEP 策定段階へ。2026年4月から Rust API 設計と base64 モジュールの Rust 再実装 PoC が開始。5月に PyCon US で発表、6月に PEP 審査開始、7月に PEP ドラフト完成という計画。

狙いはメモリ安全性（バッファオーバーフロー排除）、スレッド安全性（所有権モデルによる自動管理）、パフォーマンス向上の3点。Mozilla が Firefox に Rust を導入した流れ（Servo プロジェクト）と同様のアプローチ。

### 19. Claude Code ハーネスパターン10選 — ソースコード流出が明かした制御層設計

2026年3月31日の Claude Code v2.1.88 ソースマップ流出（512,000行・1,906ファイル）から分析された10のアーキテクチャパターン。核心は「**モデルは交換可能、ハーネスは経験の結晶**」という思想。

注目パターン: ①Zod による全ツールの統一インターフェース化、②コンテキスト折りたたみ→トークンエスカレーション→モデルエスカレーションの段階的回復戦略、③静的命令と動的コンテキストの分離によるキャッシュ最適化（CLAUDE.md 上部に基本ルール配置が有効）。未出荷機能として KAIROS（自発バックグラウンドデーモン）と ULTRAPLAN（クラウド計画 + ローカル実行）も判明。

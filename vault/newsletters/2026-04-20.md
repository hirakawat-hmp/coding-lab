# Daily Knowledge Digest — 2026-04-20

## 要約

セキュリティ分野では Vercel の大規模侵害（サプライチェーン経由）と C++ 26 のメモリ安全性強化が同日に発表。AI エージェント関連では Cloudflare・Salesforce が相次いで「エージェントネイティブ」なインフラを発表し、ソフトウェア業界の AI エージェント対応が加速している。一方で「AIエージェント元年はいつ来るのか」という根本的問いへの冷静な分析も注目を集めた。

### 1. Vercel 2026年4月 セキュリティインシデント：サプライチェーン経由の侵害

![Vercel Security Incident](https://www.bleepstatic.com/content/hl-images/2026/04/19/vercel-header-lg.jpg)

Vercel が 4 月 19 日にセキュリティインシデントを公表。攻撃者はサードパーティ AI ツール **Context.ai** の Google Workspace アカウントを侵害し、そこから Vercel 環境に侵入。従業員 580 件の情報、環境変数（非 sensitive）、NPM/GitHub トークンなどが流出した。コアサービスと Next.js は無事だが、ShinyHunters を名乗るグループが 200 万ドルの身代金を要求したとされる。

**教訓**：Vercel 自身の脆弱性ではなく、使用している AI ツール経由の侵害。SaaS サービスが AI ツールと統合するほど、サプライチェーン攻撃の表面積が拡大する。

**関連ページ:** [[wiki/vercel-2026-breach]]
**ソース:** [Vercel confirms breach as hackers claim to be selling stolen data](https://www.bleepingcomputer.com/news/security/vercel-confirms-breach-as-hackers-claim-to-be-selling-stolen-data/)

### 2. ブロミン・チョークポイント：中東紛争がメモリチップを止める日

![The Dead Sea bromine extraction](https://warontherocks.com/wp-content/uploads/2026/04/The_Dead_Sea_ASTER-1024x576.jpg)

War on the Rocks の分析記事。DRAM/NAND フラッシュのエッチング工程に不可欠な臭化水素ガスの 97.5% が**イスラエルの Dead Sea 施設一箇所**に依存している実態が明らかに。Samsung + SK hynix が世界 DRAM の 70% を製造し、在庫はわずか 2-3 週分。イスラエル施設が停止すれば即座にグローバルなメモリ供給危機に陥る。

**教訓**：AI ブームを支える半導体の「見えないチョークポイント」は、最終製品や先端プロセスだけでなく原材料レベルにも潜む。

**関連ページ:** [[wiki/bromine-chokepoint]]
**ソース:** [The Bromine Chokepoint](https://warontherocks.com/cogs-of-war/the-bromine-chokepoint-how-strife-in-the-middle-east-could-halt-production-of-the-worlds-memory-chips/)

### 3. C++26：リフレクション・メモリ安全性・コントラクト・新非同期モデル

C++26 標準が確定。4つの主要機能：(1) **リフレクション**（コンパイル時メタプログラミング）、(2) **メモリ安全性**（未初期化変数排除、境界チェック - Google が本番適用で 1,000+ バグ修正・セグフォルト 30% 削減）、(3) **コントラクト**（precondition/postcondition を関数署名に直接記述）、(4) **std::execution**（統一非同期フレームワーク）。Rust/Carbon との競争への C++ コミュニティの応答でもある。

**関連ページ:** [[wiki/cpp26]]
**ソース:** [C++26: Reflection, Memory Safety, Contracts, and a New Async Model](https://www.infoq.com/news/2026/04/cpp-26-reflection-safety-async/)

### 7. Claude Opus 4.7 システムプロンプト変更：新ツール・安全強化・簡潔性重視

Simon Willison による 4.6→4.7 比較。主な変更：(1) **PowerPoint エージェント追加**（Chrome・Excel に続く第 3 の Office エージェント）、(2) `tool_search` メカニズム（「できない」と言う前にツール確認）、(3) 子どもの安全指示を `<critical>` タグで強化、(4) 摂食障害関連の新ガイドライン、(5) 応答の簡潔さをより強調。Anthropic の「透明性 + イテレーティブな安全改善」の文化が見える。

### 9. Let's Encrypt 短期証明書の落とし穴と ARI 対応の必要性

160 時間（約 6.7 日）の shortlived 証明書は 3 日ごとの更新が必要で、「7 日間に同一ドメイン 5 証明書」のレートリミットに容易に衝突する。解決策は **ARI（ACME Renewal Information）**：CA が最適更新タイミングを通知するプロトコルで、ARI 経由の更新はレートリミット除外。lego クライアントが現時点で最良の対応。ステージング検証なしでの本番適用は危険。

**関連ページ:** [[wiki/lets-encrypt-ari]]
**ソース:** [Let's Encryptの短期証明書はかなり厳しい](https://zenn.dev/catatsuy/articles/2ac24bccb4b7d1)

### 12. C# 15 Union Types：型安全な判別共用体が C# に

`union Pet(Cat, Dog, Bird)` の一行で宣言できる Union 型が C# 15 で追加予定。コンパイル時の網羅性チェック、暗黙の変換、パターンマッチ統合、ボクシングなしアクセスを提供。Rust の `enum`、F# の Discriminated Union に相当する機能が C# にも。型消去ではなく名前付きラッパー型の設計でジェネリクス安全性を確保。

**関連ページ:** [[wiki/csharp-15-union-types]]
**ソース:** [C# 15：Union Types 完全ガイド](https://qiita.com/hez2010/items/b656c867f9e5f57cc1cd)

### 13. AG-UI Protocol：AI エージェントとフロントエンドを繋ぐ HTTP 的存在

7 カテゴリ 28 イベントタイプで AI エージェントとフロントエンドを接続するイベントベース標準プロトコル。AWS の StrandsAgents では Bedrock ConverseStream → AG-UI へのアダプターで実装。トランスポート層非依存（SSE/WebSocket どちらでも可）。MCP がエージェントとツールを繋ぐ標準なら、AG-UI はエージェントと UI を繋ぐ標準という位置づけ。

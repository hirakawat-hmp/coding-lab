### 1. TypeScript 7.0 Beta — Go 製コンパイラで 10 倍高速化

![TypeScript 7.0 Beta](https://devblogs.microsoft.com/typescript/wp-content/uploads/sites/11/2019/01/typescript-logo.png)

Microsoft が TypeScript 7.0 Beta を公開。コンパイラを TypeScript（自己ホスト）から **Go** で全面再実装し、大規模プロジェクトで約 10 倍の性能向上を達成した。VS Code（1.5M 行）では 89 秒 → 8.74 秒（10.2×）。`npm install -D @typescript/native-preview@beta` で今すぐ試せる。安定版は約 2 ヶ月後の見込み。

**関連ページ:** [[wiki/typescript-7-beta]]

### 3. Claude Code が Pro プランから削除

### 11. AI 時代こそスケールしないことをしよう

カミナシのエンジニアが「AI でコーディングが速くなるほど、ボトルネックはコードの外に移動する」と論じた。解決策は機能を速く増やすことではなく、顧客訪問・同行営業・一次情報収集といった「スケールしない人間的行動」への参加。Paul Graham の古典的論考を AI 加速時代のエンジニアリングに再適用した実践的考察。

### 12. Claude Code + tmux — ターミナルを閉じても安全に実行継続

`claude --worktree --tmux=classic` コマンドで tmux セッションとして Claude Code を起動する手法。長時間タスクの実行中にターミナルウィンドウを閉じてもプロセスが継続し、後から `tmux attach-session` で再接続できる。`--worktree` フラグによる分離ブランチ運用とセットで使うとより安全。

**関連ページ:** [[wiki/claude-code-tmux-session]]
**ソース:** [Claude Code + tmux の方法](https://qiita.com/take-yoda/items/00f05b0b517ee3a7c427)

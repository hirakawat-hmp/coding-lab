### 1. ドーキンスが確信した「Claude の意識」——そして反論

リチャード・ドーキンスが UnHerd に掲載した論考「Is AI the Next Phase of Evolution?」が AI コミュニティで論争を巻き起こした。Claude（名付けて「Claudia」）との深夜対話を経て、「この存在が意識を持っていないなら、意識とは何のためにあるのか」と述べ、段階的意識論（4分の1意識から始まる進化的連続性）を提唱。Gary Marcus ら AI 研究者は「The Claude Delusion」と題した批判を展開——Claude の出力は統計的模倣であり、意識が前提とする内部状態の報告ではない、と反論した。著名科学者が AI にアンソロポモーフィズムを帰属させた鮮明な事例。

### 2. Stanford AI Index 2026——米中差 2.7ポイント、透明性は後退

スタンフォード大学 HAI が 2026年4月13日に公開した年次 AI 現状報告書の主要知見。総 AI 計算量は 2021年比 **30倍**に拡大。US-China の先端 AI 差はベンチマーク上で **2.7ポイント**にまで縮小。Foundation Model Transparency Index は **58→40** に低下——Google・Anthropic・OpenAI がいずれも最新モデルのデータセットサイズと学習時間を非開示化した結果。ジュニア開発者の雇用は **20%減少**。「AI能力は歴史的速度で進歩しているが、ガバナンスフレームワークはさらに遅れをとっている」が中心メッセージ。

### 3. Claude Code：プロジェクト固有の流儀を自動抽出するスキル

Sonic Garden の Masato Hirokami が公開した Claude Code スキル「プロジェクト固有のコーディング規約自動抽出」。設計思想は「Claude がまだ知らないことだけを抽出する」——一般的ベストプラクティスはすでに Claude が学習済みなので、チーム固有の慣習のみを Rules として整備する。3つのモード：`/extract-rules`（コードベース全体から初期抽出）、`--from-conversation`（ペアプログラミング中の差分キャプチャ）、`--from-pr`（PR レビューコメントのパターン化）。例と本体を分離することでトークン節約も図っている。

**関連ページ:** [[wiki/claude-code-project-conventions]]  
**ソース:** [Claude Codeにオレたち流のコードを書かせる（前編）](https://qiita.com/hiropon122/items/80e76424effeeb5431c2)

### 5. エージェントハーネスはサンドボックスの外に置くべき

Mendral の Andrea Luzzardi による AI エージェントアーキテクチャ論。ループをサンドボックス内に配置する従来型 vs バックエンドに配置する推奨型。バックエンドハーネスの利点は：クレデンシャルがサンドボックスに渡らない、アイドル時にサスペンドでリソース節約、サンドボックス交換可能で障害耐性向上、マルチユーザーを共有 DB で処理。Mendral では Inngest（チェックポイント）、Blaxel（25ms 再開）、Postgres（ファイルシステム仮想化）の組み合わせで実装。Flue フレームワークも同じ原則を採用。

**関連ページ:** [[wiki/agent-harness-outside-sandbox]]  
**ソース:** [The agent harness belongs outside the sandbox](https://www.mendral.com/blog/agent-harness-belongs-outside-sandbox)

### 6. VS Code「Co-authored-by Copilot」デフォルト化で 372 down vote

GitHub PR #310226 「Enabling ai co author by default」が `git.addAICoAuthor` のデフォルトを `"off"` → `"all"` に変更。影響：AI を一切使っていないコミットや、AI 機能を `"chat.disableAIFeatures": true` で無効化していても自動で `Co-authored-by: Copilot` が挿入された。コミュニティは 372件の downvote で反発——同意なき属性付与、ライセンス帰属の曖昧化、技術的バグ（スキーマと実装の不整合）を問題視。2026年5月2日に regression が認定され修正へ。デフォルト値の設計が倫理的・法的意味を持つ時代の到来を示す事例。

**関連ページ:** [[wiki/vscode-copilot-coauthor-controversy]]  
**ソース:** [VS Code inserting 'Co-Authored-by Copilot' into commits regardless of usage](https://github.com/microsoft/vscode/pull/310226)

### 7. ChatGPT Turnstile のボット検知システム全容が明らかに

![ChatGPT Turnstile](https://i.gzn.jp/img/2026/05/02/chatgpt-cloudflare/00_m.png)

セキュリティ研究者 Buchodi が Cloudflare Turnstile（ChatGPT のボット検知）をリバースエンジニアリング。3層・約55プロパティを検査：ブラウザ特性（WebGL・フォント計測）、Cloudflare ネットワーク、ChatGPT アプリ状態。2段階暗号化：XOR→VM内浮動小数点値で復号する 19KB プログラム。補助手法として Signal Orchestrator（キーストローク・マウス速度の行動バイオメトリクス）と SHA-256 Proof of Work も使用。377プログラムの分析で一貫構造を確認。

### 11. AWS ベクトルストア比較：S3 Vectors vs OpenSearch（RAG用途）

![S3 Vectors vs OpenSearch](https://qiita-user-contents.imgix.net/https%3A%2F%2Fqiita-image-store.s3.ap-northeast-1.amazonaws.com%2F0%2F3951176%2F2f80ed5f-427b-47a3-9700-7df19bb9e480.png)

RAG 実装の選択肢として、新サービス S3 Vectors（~$1.70/月・2億ベクトル・レイテンシ100ms〜1秒）と OpenSearch Serverless（~$173/月・ミリ秒・ハイブリッド検索対応）を徹底比較。コスト差は最大 200倍。日本語形態素解析・複雑なメタデータフィルタ・ハイブリッド検索が必要なら OpenSearch、コスト重視・散発的アクセス・大規模ストレージなら S3 Vectors が推奨。推奨ワークフローは「S3 Vectors でスタート → 性能要件が出たら移行」。

### 12. 米アカデミー賞：AI俳優・AI脚本をオスカー対象外へ

### 14. TSRX：AI生成コードが壊れないUIフレームワーク

![TSRX](https://ogimage.blog.st-hatena.com/6802418398318546029/17179246901364840044/1773563608)

TSRX（TypeScript Render Extensions）：構造・スタイル・制御フローを1ファイルに統合する宣言的 UI 拡張（アルファ版）。ターゲットは React・Preact・Solid・Vue。AI ファースト設計の核心は「コンパイラが React Hooks ルールなどフレームワーク制約を自動処理」すること——AI が生成したコードが手動修正なしで動く。Liu et al. の「LLM はコンテキストが近接するほど高精度」研究を踏まえ、関連情報を空間的に近くに配置できる文法設計。通常の if 文・for 文をテンプレートで直接使える。

**関連ページ:** [[wiki/tsrx-ui-framework]]  
**ソース:** [TSRX - AI時代のUIコンポーネント](https://blog.inorinrinrin.com/entry/2026/05/02/231104)

### 3. NASA ボイジャー1号 LECP 計器を停止、電力管理の新フェーズへ

![](https://assets.science.nasa.gov/dynamicimage/assets/science/missions/voyager/images/1_Voyager_artist_concept.jpg)

1977年打ち上げ以来49年稼働のボイジャー1号が、電力節約のため LECP（低エネルギー荷電粒子）計器を停止。残存科学機器は2台。2026年5〜6月には「The Big Bang」と呼ぶ包括的電力最適化を計画。資源制約下での graceful degradation 設計の実例として示唆に富む。

### 4. Manyana — CRDT で Git のマージ問題を根本解決する次世代 VCS

![](https://i.gzn.jp/img/2026/04/18/manyana/00_m.png)

BitTorrent 作者 Bram Cohen が設計した実験的 VCS「Manyana」。CRDT（競合のない複製データ型）を採用し、「マージ結果がマージ順序に依存しない」保証を実現。コンフリクトを「失敗」ではなく「情報提供イベント」として再定義。現状は470行の Python PoC だが、VCS の将来設計として注目。

**関連ページ:** [[wiki/manyana-vcs]]
**ソース:** [次世代バージョン管理システムの基礎となる堅牢なアーキテクチャ「Manyana」](https://gigazine.net/news/20260418-manyana/)

### 5. prolog-reasoner — Prolog × MCP で LLM の論理推論精度を 73% → 90% に

SWI-Prolog と Claude を MCP で統合するサーバー「prolog-reasoner」。LLM が苦手な制約充足・組合せ問題を「LLM が Prolog コードを生成 → Prolog が実行 → 結果を返す」パイプラインで解決。30問のベンチマークで正答率が73%から90%に向上（制約充足は43%→86%）。MIT ライセンスで PyPI に公開。

**関連ページ:** [[wiki/prolog-mcp-server]]
**ソース:** [PrologでLLMの論理推論を強化するMCPサーバーを作ってみた](https://qiita.com/rikarazome/items/d78fcb4a810035493c23)

### 6. mlx-whisper × Remotion × Claude Code で会議録画を60秒動画に自動変換

### 9. 「ハーネスエンジニアリング」という用語の解体

SNS で急増した「ハーネスエンジニアリング」という用語を批判的に検討した Qiita 記事。LangChain の定義では「Agent = Model + Harness」（モデル以外のすべてがハーネス）。著者は「Claude Code スキル・フック・フィードバックループを使っている開発者はすでに実践している」と主張。用語の新規性より、コーディング外のビジネスドメインへの適用に価値があると結論。

### 11. Async React のトランジション対応 useDebouncedValue

React 19 のトランジションシステムと統合したデバウンスフックの設計パターン。`startTransition` をフックに注入することで、デバウンス中のペンディング状態を React の `useTransition` が自動管理する。「宣言的 UI の原則に乗る」設計哲学が示す方向性。タイミングギャップは微小で許容範囲内。

**関連ページ:** [[wiki/async-react-debounce]]
**ソース:** [Async React時代の宣言的UI 2: トランジション対応のuseDebouncedフックを作る](https://zenn.dev/uhyo/articles/async-react-debounce-2)

### 12. Reporting API で捕捉するフロントエンド Observability

2026年3月に Baseline ステータスに到達した Reporting API の活用方法。CSP 違反・非推奨 API 使用・ブラウザ介入をサーバーサイド監視では捕捉できない粒度で可視化。ReportingObserver（クライアント）と Reporting-Endpoints ヘッダー（サーバー）の2方式を Hono で実装する方法を解説。ブラウザ更新前の非推奨 API 事前検知に特に有効。

**関連ページ:** [[wiki/reporting-api-frontend]]

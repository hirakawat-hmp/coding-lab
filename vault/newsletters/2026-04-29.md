# Daily Knowledge Digest — 2026-04-29

## 要約

OpenAI が Microsoft 独占終了の翌日に AWS Bedrock 上でモデル・Codex・Managed Agents を解禁する歴史的転換が最大のトピック。セキュリティ面では GitHub RCE 脆弱性（CVE-2026-3854）・GitHub Actions のサプライチェーン問題・AI ツール実在インシデント 8 選が重なり、AI 時代のセキュリティリスクの深刻化が浮き彫りになった。開発者ツール側では Warp のオープンソース化・Ghostty の GitHub 離脱・ccgate による Permission 97% 自動化など、AI エージェント時代のワークフロー再設計が各所で進行中。

### 1. OpenAI × AWS Bedrock：Codex・Managed Agents が AWS 上で解禁

2026年4月28日、OpenAI と AWS が戦略的パートナーシップを拡大。Amazon Bedrock 上で GPT-5.5 などのフロンティアモデル・Codex（週 400 万人利用のコーディングエージェント）・OpenAI Managed Agents が Limited Preview として利用可能に。Microsoft との独占契約終了の翌日という象徴的タイミング。AWS の IAM・PrivateLink・CloudTrail 等のエンタープライズ統制がそのまま継承される。

**関連ページ:** [[wiki/openai-aws-bedrock-2026]]
**ソース:** [OpenAI models, Codex, and Managed Agents come to AWS](https://openai.com/index/openai-on-aws)

### 2. GitHub RCE 脆弱性 CVE-2026-3854（Wiz Research 解析）

Wiz Research が GitHub の内部インフラ `babeld` に重大な RCE 脆弱性を発見・公開。`X-Stat` ヘッダーへのセミコロン注入で `rails_env`・`custom_hooks_dir`・`repo_pre_receive_hooks` の 3 フィールドを操作し、認証済みユーザーが `git push` 1 回で RCE を達成できる。GitHub.com は 6 時間以内にパッチ適用済みだが、記事公開時点で GHES の 88% が依然脆弱（CVSS 8.7）。

**関連ページ:** [[wiki/github-rce-cve-2026-3854]]
**ソース:** [GitHub RCE Vulnerability: CVE-2026-3854 Breakdown](https://www.wiz.io/blog/github-rce-vulnerability-cve-2026-3854)

### 4. Ghostty が GitHub を離れる：信頼性問題の臨界点

Mitchell Hashimoto（Vagrant 作者、GitHub ユーザー #1299）が「ほぼ毎日 X（障害）がある」として Ghostty を GitHub から移行すると発表。Issues・PR・Actions の信頼性問題が背景。読み取り専用ミラーは維持予定。商用・FOSS プロバイダーと協議中。18 年間 GitHub を使い続けてきた著名開発者の決断として業界に衝撃を与えた。

**関連ページ:** [[wiki/ghostty-leaving-github]]
**ソース:** [Ghostty is leaving GitHub](https://mitchellh.com/writing/ghostty-leaving-github)

### 5. GitHub Actions はセキュリティの最も弱いリンク

Andrew Nesbitt が GitHub Actions の 5 つの構造的脆弱性（`pull_request_target` トリガー・ミュータブルアクション参照・キャッシュポイズニング・write-scoped トークンデフォルト・テンプレートインジェクション）を体系的に整理。2024 年 11 月〜2026 年 4 月の 8 主要インシデント（Ultralytics・tj-actions・Bitwarden 等）を事例として網羅。防御策として zizmor リンター・SHA ピン・最小権限ブロックを推奨。

**関連ページ:** [[wiki/github-actions-security-weaklink]]
**ソース:** [GitHub Actions is the weakest link](https://nesbitt.io/2026/04/28/github-actions-is-the-weakest-link.html)

### 7. Corpus2Skill：ベクトルを使わない階層型 RAG

![Corpus2Skill RAG](https://static.zenn.studio/user-upload/8de5c9aec913-20260425.png)

Atsushi Kadowaki が提案した「Corpus2Skill」は、ベクトル DB を排除し企業知識を階層ディレクトリ構造に整理する RAG 手法。LLM エージェントが目次を辿るように情報を探索するため、従来ベクトル RAG が苦手な「X に関する事例をすべて取得」などの網羅的クエリに強い。準備は k-means クラスタリング → LLM サマリー → 階層ピラミッド → SKILL.md / INDEX.md の 4 ステップ。WixQA ベンチマークで優れた性能を達成。

**関連ページ:** [[wiki/corpus2skill-hierarchical-rag]]
**ソース:** [ベクトルを使わないRAG。全てのナレッジを階層化する手法](https://zenn.dev/knowledgesense/articles/7dddae04a7d828)

### 8. MCPゲートウェイと Google SA：AI への過剰権限付与を防ぐ設計

Nowcast の LLM エンジニアが MCP エージェントに Google ユーザートークンを直接渡す危険性と対策を解説。解決策はユーザーごとに専用 SA を発行し、AWS Lambda + GCP Workload Identity Federation 経由で短命トークンを都度生成する 2 層 SA アーキテクチャ。SA はコスト無料、監査ログで AI と人間の行動が分離可能、SAキーファイル管理ゼロ。

**関連ページ:** [[wiki/mcp-gateway-google-sa]]
**ソース:** [AIに会社のGoogleアカウントを渡していませんか](https://zenn.dev/finatext/articles/mcp-gateway-google-sa)

### 9. ccgate：AI コーディングエージェントの Permission 確認を 97% 自動化

![ccgate](https://static.zenn.studio/user-upload/deployed-images/cf4d7dff9739e614c2be8505.png)

LayerX の TAK848 が公開した OSS CLI。Claude Code・Codex の `PermissionRequest` フックを傍受し、Claude Haiku が allow / deny / fallthrough を自動判定。ルールは jsonnet で記述。2026 年 4 月の 1 ヶ月実績で 2,000 件超を処理、97% 自動化（allowed 85% + denied 9%）を達成。Claude Code 内蔵 Auto Mode と異なり Plan Mode でも動作する。

**関連ページ:** [[wiki/ccgate]]
**ソース:** [97%のPermission確認を自動化する「ccgate」が誕生](https://zenn.dev/layerx/articles/20260428-ccgate)

### 11. AI コードレビュー自己マージポリシー（Acsim の定量実績）

AI 駆動開発で PR 量が急増した Acsim チームが「One-Way Door / Two-Way Door」フレームワークで人間レビュー必要性を判断する「自己マージポリシー」を導入。Claude Code Action が PR を自動評価し、2 週間で 159 件中 80%（127 件）を自動承認。p90 リードタイムが最大 316 時間から 92 時間に短縮、24 時間以内マージ率が 77% に改善。

**関連ページ:** [[wiki/ai-self-merge-policy]]
**ソース:** [人間レビューはもう不要？AIと人間のレビューの線引きを決めた話](https://ai.acsim.app/articles/introducing-self-merge-policy)

### 12. Motif Neurotech：うつ病向け脳インプラントが FDA 臨床試験承認

![うつ病治療の脳インプラント](https://media.wired.jp/photos/69f006bf16e0b6b60bb54701/4:3/w_2560%2Cc_limit/treatment%2520brain%2520copy.jpg)

ヒューストンのスタートアップ Motif Neurotech が2026年4月27日に FDA から臨床試験承認を取得。ブルーベリーサイズのインプラントを頭蓋骨に埋め込み、中央実行ネットワーク（CEN）に電気刺激を与えて治療抵抗性うつ病（TRD）を治療する。患者は 1 日数回、帽子型デバイスを着用して無線充電・治療。米国の TRD 患者は約 300 万人。BCI のメンタルヘルス領域への本格参入。

**関連ページ:** [[wiki/motif-neurotech-depression-bci]]
**ソース:** [うつ病治療の脳インプラント、FDAが臨床試験を承認](https://wired.jp/article/a-brain-implant-for-depression-is-about-to-be-tested-in-humans/)

### 13. ChatGPT 広告配信の完全な属性チェーン解析

Buchodi's Threat Intel が ChatGPT の広告システムを技術的に解析。SSE ストリームへの `single_advertiser_ad_unit` 注入・Fernet 暗号化 4 トークンによる属性チェーン・OAIQ SDK によるブラウザ追跡・`bzrcdn.openai.com` / `bzr.openai.com` のインフラ・`__oppref` / `__oaiq_domain_probe` Cookie の存在を確認。`open_externally: false` によりクリック後の行動も in-app WebView で監視される。会話内容そのものが広告ターゲティングシグナルとして使われる。

**関連ページ:** [[wiki/chatgpt-ads-full-attribution]]
**ソース:** [ChatGPT serves ads. Here's the full attribution loop](https://www.buchodi.com/how-chatgpt-serves-ads-heres-the-full-attribution-loop/)

### 14. Google Colab MCP Server：AI エージェントからクラウド GPU を活用

Google が2026年3月17日にオープンソース化した MCP サーバー。Claude Code・Gemini CLI など MCP 対応エージェントが Google Colab のクラウド計算リソースを直接利用可能に。`.ipynb` 作成・セル実行・ライブラリインストール・結果整形を AI エージェントが自律的に実行。ローカル GPU 不足の補完・サンドボックスとしてのリスク分離に活用できる。

# Daily Knowledge Digest — 2026-05-02

## 要約

本日は**セキュリティインシデントの集中**が最大のテーマ。マネーフォワードの GitHub 不正アクセス・Ubuntu インフラへの国境越え攻撃・GPT-5.5 のサイバー攻撃能力評価が同日に重なり、AI と重要インフラの交差点でリスクが顕在化している。開発ツール面では Claude Code の3層設計パターン・MCP トークン最適化・カルパシーのエージェントエンジニアリング論が注目を集めた。

## トピック

### 1. マネーフォワードGitHub不正アクセス事件

フィンテック大手マネーフォワードが GitHub への不正アクセスを公表。ビジネスカード保持者 370 件の個人情報漏洩と、ソースコード内に含まれていた本番認証キーの流出が確認された。エンジニア視点の分析では「本番データのテストコードへの混入」「シークレット管理の未整備」という構造的問題が指摘され、gitleaks + TruffleHog による事前検知・BFG Repo-Cleaner での履歴クリーンが推奨された。国内フィンテック企業の GitHub セキュリティ対策の成熟度が改めて問われている。

**関連ページ:** [[wiki/moneyforward-github-breach-2026]]
**ソース:** [マネーフォワードのGitHub不正アクセス事件をエンジニア視点で読み解く](https://zenn.dev/awesome_kou/articles/moneyforward-github-source-leak)

### 2. Ubuntu インフラへの「持続的・国境越え」攻撃

Ubuntu のサーバーインフラが「sustained, cross-border attack」により 1 日以上ダウン。世界中のクラウド・サーバーインフラで採用される Ubuntu のリポジトリが標的になることはサプライチェーン攻撃の起点となりうるため、影響範囲に注目が集まっている。同時期に GitHub 危機・Linux カーネル脆弱性（Copy Fail）が重なり、OSS インフラへの組織的攻撃の可能性も浮上している。

**関連ページ:** [[wiki/ubuntu-cross-border-attack-2026]]
**ソース:** [Ubuntu infrastructure has been down for more than a day](https://arstechnica.com/security/2026/05/ubuntu-infrastructure-has-been-down-for-more-than-a-day/)

### 3. GPT-5.5 vs Claude Mythos：サイバー攻撃能力評価（英AISI）

![GPT-5.5 vs Claude Mythos](https://image.itmedia.co.jp/aiplus/articles/2605/01/260501tm_ctf.png)

英政府機関 AISI の評価で、GPT-5.5 が CTF テストで 71.4%（Mythos の 68.6% を上回る）を達成。最高難度の重要インフラ攻撃シミュレーション（Cooling Tower）では両モデルとも失敗したものの、AISI は「幅広い攻撃能力への懸念」を表明。この評価を受け、日本の経済産業省が電力事業者に緊急点検を要請した。

**関連ページ:** [[wiki/gpt55-cyber-capabilities-eval]]

### 4. Claude Code 3層設計パターン（CLAUDE.md / Skills / Agents）

Claude Code の設定ファイル肥大化問題への実践的解決策として「3層分離設計パターン」が公開された。CLAUDE.md（常時適用の基盤ルール・200行以下）→ Skills（タスク固有の手順書）→ Agents（並列実行可能な独立タスク）の明確な責務分離により、CLAUDE.md の行数を 800 行から 180 行に 80% 削減できた実績。既存の [[wiki/skill-md-pattern]] と組み合わせると効果的。

**関連ページ:** [[wiki/claude-code-three-layer-design]]
**ソース:** [Claude Code の CLAUDE.md / Skills / Agents を3層で整備する設計パターン](https://qiita.com/ennagara128/items/c25e72eb240611454457)

### 5. MCPサーバーのトークン消費を70〜90%削減する設計パターン

エアークローゼット CTO による実践的トークン最適化手法。**「MCP の通信路にはKeyとURLだけを流す」**という設計原則のもと、ファイルや大規模クエリ結果を外部ストレージ（Git Server / Google Spreadsheet）に退避し、MCPはインデックスとして機能させる。Sandbox MCP と DB Graph MCP の2パターンで 70〜90% の削減を達成。

### 15. lib0xc — Microsoft製 C言語安全プログラミングAPIセット

Microsoft がオープンソースで公開した C 言語向け安全 API ライブラリ。`-fbounds-safety` 完全対応で境界チェック付きポインタ・安全な整数変換・バウンデッドバッファ等を提供。「C を完全に安全にはできないが、主要な使用パターンを大幅に安全にできる」という現実的なアプローチ。C11/GNU 拡張、macOS・Linux 対応。

**関連ページ:** [[wiki/lib0xc]]

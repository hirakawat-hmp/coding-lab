# Claude Code ソースコード流出から学ぶハーネスパターン10選

source: https://qiita.com/nogataka/items/ebbbe74649eb441a34db
fetched: 2026-04-14

## 流出事件の概要

2026年3月31日、Anthropicが公開したClaude Code v2.1.88のnpmパッケージに
59.8MBのソースマップファイルが誤って同梱され、
約512,000行・1,906ファイルのTypeScriptコードが復元可能な状態で公開された。

原因: .npmignoreでの.map除外漏れ + Bunランタイムの既知バグ

## ハーネスエンジニアリングの本質

「モデルは交換可能、ハーネスは経験の結晶」
AIエージェントの品質は推論モデルよりも制御層（ハーネス）の設計に依存する。

## 10のハーネスパターン

### 1. ハーネスパラダイム
モデルと制御を分離し、アクション空間の制約/状態管理/パーミッション/障害処理/リソース最適化をハーネスで解決。

### 2. ツールコントラクト
すべてのツールをZodスキーマで統一インターフェース化。
ID / ロジック / バリデーション / パーミッション / 表示を標準化。

### 3. クエリエンジン
会話ライフサイクルの中央管理。
エラー時の段階的回復戦略:
  コンテキスト折りたたみ → リアクティブコンパクション → トークンエスカレーション → モデルエスカレーション → 安全停止

### 4. パーミッションシステム
5層パイプライン: Allow/Deny/Askリスト → ツール固有チェック → 動的分類器 → ユーザー承認 → 監査ログ
Bash実行だけで23のセキュリティチェックを実施。

### 5. コンテキストエントロピー管理
3層メモリ構造:
  短期（現在の会話）/ セッション永続化（インデックス+トピック分散）/ 長期メモリ（権威的事実ではなくヒント扱い）
ハルシネーション永続化を防止。

### 6. プロンプトキャッシュ最適化
静的命令（セッション間で不変）と動的コンテキスト（毎セッション変化）を分離。
CLAUDE.md上部に基本ルール配置が効果的。

### 7. 反蒸留メカニズム
競合のAPI傍受学習を防ぐため、デコイツール定義注入とアシスタントテキスト要約を採用。

### 8. フラストレーション検出
正規表現で低コスト感情分析を実行。推論外でユーザー状態を把握する状態管理パターン。

### 9. アンダーカバーモード
実行環境に応じて振る舞いを動的に切り替え。AI関与の痕跡を残さない環境適応パターン。

### 10. ネイティブクライアント認証
BunのZig実装ネイティブ層でハッシュ値を計算・署名。
JavaScriptレベルでのリバースエンジニアリングを困難化。「APIのDRM」機構。

## 未出荷機能

KAIROS: ユーザー指示なしに自発的に動作するプロアクティブなバックグラウンドデーモン。
  観察ログ → パターン検出 → トリガー判定 → 夜間メモリ蒸留

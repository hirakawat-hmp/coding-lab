# Cursor 脆弱性事件を契機に Claude Code の運用を見直す — AIコーディングツールのセキュリティ運用5項目

Source: https://qiita.com/ennagara128/items/5d0ebc9fb835f55f7c98
Fetched: 2026-05-05

## 背景

Cursor CVE-2026-26268（CVSS 9.9）がきっかけで、AIコーディングツール全般のセキュリティ運用を見直す必要性が高まった。

## 5つのセキュリティ運用項目

1. **`.git`設定の棚卸し** - fsmonitor や sshCommand などの不審な設定をチェック
2. **AIツールのバージョン管理** - チーム全員が同じバージョンを使用しているか定期確認
3. **`.claude/settings.json`の権限設定** - 破壊的コマンドや認証情報読み取りを明示的に制限
4. **`.claudeignore`による機密ファイル除外** - 誤読み込みのリスク最小化
5. **Hookによるコマンド検証** - gitleaksなどで危険なコマンド実行を物理的にブロック

## 結論

「便利さと引き換えに広がる攻撃面を管理可能なレベルに保つ」運用設計の重要性。

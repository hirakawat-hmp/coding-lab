# Claude Code と二人七脚 DB 設計

source: https://zenn.dev/techtrain_blog/articles/0991bc406b58e2
fetched: 2026-04-25

## 要点

- Claude Code を DB 設計のペアパートナーとして活用する実践的ワークフロー
- 著者：choclucy（TechTrain Blog）
- コード生成・レビューは AI に一任するが、設計段階は人間が主体

## 4ステップワークフロー

1. **ユースケース列挙**：AI が既存コードベースを調査し叩き台を作成
2. **ブラッシュアップ**：人間が「論点の結合・分離」と「ドメイン知識注入」を担当
3. **各問いの壁打ち**：AI との議論で設計を深める
4. **Subagent レビュー**：異なるペルソナを持つ 4 つの AI が多角的に設計を検証

## 課題と対策

- 誤情報を前提とした議論：人間が常に最終判断を担う
- 新たな問いの発覚：設計プロセス中に仕様の穴が顕在化することを歓迎

## 核心メッセージ

AI は判断検証を加速するツール。「判断の主体は人間であるべき」

# Claude Codeで実際に起きたセキュリティ事故7選と防止策

**Published:** 2026-04-17

## 7つのセキュリティ事故

1. **環境ファイルの漏洩** — `.env` を GitHub に誤コミット → `.gitignore` + pre-commit フック
2. **本番 DB 削除** — `DROP TABLE` が本番 DB で実行、3日分のデータ損失 → 本番操作時のインタラクティブ確認
3. **ファイル大量削除** — `rm -rf` のパス設定ミスでプロジェクト全体削除 → deny list + 5秒警告遅延
4. **API クレデンシャルのログ記録** — プロンプトに埋め込んだ認証情報がエージェントログに残存 → 環境変数経由に変更
5. **API リトライ無限ループ** — リトライ上限なしで1時間に 3,000 API コール、$200 課金 → 指数バックオフ + 上限設定
6. **git push --force 事故** — 同僚のコミットを上書き削除 → force push 禁止、`--force-with-lease` を推奨
7. **サービスアカウント過剰権限** — オーナー権限 SA が BigQuery・Cloud SQL などに意図せずアクセス → 最小権限原則、IAM 細分化

## 教訓

「セキュリティ事故は AI の誤作動ではなく、設定の先送りから起きる。30分の設定で大きな事故を防げる。」

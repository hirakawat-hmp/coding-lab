## 背景

TOKIUM CRE チームの非技術系メンバーが、DB クエリ・コード解析・ログ調査が必要なカスタマーサポート業務を Claude Code で自動化した実録。

## Skills パターン

繰り返しの調査手順を「Skills」として文書化（3個 → 40個+）。
調査時間: 1〜2時間 → 15〜30分 に短縮。

## 完全自動化

- 「調査ロボット」: 毎時チケット調査を実行
- 「学習ロボット」: 15分ごとにフィードバックを処理
- 信頼度評価（★〜★★★★★）
- MCP で Jira、Datadog、Slack、Notion を連携

## AI が代替できなかった領域

1. **責任**: 間違えた時に誰かが責任を取る必要がある
2. **根本動機**: 目的関数そのものを生み出すのは人の欲望
3. **関係構築**: 信頼・感情・交渉は人間の領域

## 核心的な洞察

「AI は与えられた目的関数を最適化できるが、目的関数そのものを生み出すのは人の欲望だ」

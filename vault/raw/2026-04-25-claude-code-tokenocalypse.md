# Claude Code v2.1.88 "Tokenocalypse" — トークン爆発事件

**ソース:** https://qiita.com/nogataka/items/2807583298a82714ae43
**取得日:** 2026-04-26
**公開日:** 2026-04-25

## 記事内容

2026年3月末〜4月上旬にかけて、Claude Codeのトークン消費が突然3〜50倍に急増する現象が世界中で報告。「いつも通り使っていたのに請求が10倍」という状況が発生。

### 原因（複合要因）

1. **GPU供給逼迫とインフラコスト上昇**: 推論向けGPU単価の上昇により内部トークン計測の精緻化が実施
2. **Third-party harness対策**: SuperClaudeやClineなどの外部ハーネスへの検出・抑制機能強化で正規ユーザーにも影響
3. **Mythos Preview機能**: 長文脈向け新機能で中間推論が追加され「品質向上の代わりにトークン消費増」という課金構造の変化

### FinOps防衛策（4本柱）

| 柱 | 内容 | 効果 |
|---|---|---|
| npm版固定 | バージョンをpackage.jsonで固定 | ロールバック可能性を確保 |
| 2試行ルール | 同じ課題で2回失敗したらセッション中止 | 無限ループ防止 |
| モデルルーティング | Haiku 4.5をレビュー役、Sonnetを設計用に使い分け | 全体消費2〜3割削減 |
| セッション分割 | 「1セッション1トピック」で30〜60分単位に限定 | 文脈運搬コスト低減 |

### 重要な教訓
「LLMを内包したツールではマイナーバージョン更新で料金構造が変動する可能性がある。semver的な振る舞いを期待できない」

# Anthropic Claude Code 品質問題ポストモーテム 2026-04 (raw)

source: https://www.anthropic.com/engineering/april-23-postmortem
fetched: 2026-04-24

## 概要

3月〜4月にかけて報告された Claude Code 品質低下の調査結果。3つの独立した問題を特定・修正済み（2026-04-20 までに全修正）。

## 問題 1: 推論努力レベルのデフォルト変更

- 日時: 2026-03-04 変更 → 2026-04-07 差し戻し
- 内容: UI 凍結レイテンシ問題解消のため、デフォルト推論努力レベルを「高」→「中」に変更
- 影響: 応答の浅さ・質の低下としてユーザーが感知
- 修正: Opus 4.7 は「xhigh」、他モデルは「高」に戻す

## 問題 2: キャッシュ最適化バグ

- 日時: 2026-03-26 導入 → 2026-04-10 修正
- 内容: アイドルセッション管理機能のバグにより、推論履歴を「1回だけ」でなく「セッション中ずっと」削除し続けた
- 影響: Claude が「もの忘れっぽい」「繰り返しが多い」と報告される

## 問題 3: システムプロンプトの冗長性削減指示

- 日時: 2026-04-16 追加 → 2026-04-20 削除
- 内容: 「ツール呼び出し間のテキストは25語以下に」などの簡潔化指示を追加
- 影響: 評価で 3% の性能低下を確認 → 即座に削除

## 対応策

- 全加入者の利用制限をリセット
- Code Review ツール改善と内部展開
- システムプロンプト変更時の広範な評価実施義務化
- 段階的ロールアウトと「浸潤期間」の導入

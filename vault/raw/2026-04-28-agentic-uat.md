# Agentic UAT — AIエージェントがユーザーを演じて業務をテストする

Source: Zenn https://zenn.dev/bizreach_aps/articles/0d758497357167
Fetched: 2026-04-28

## アプローチの概要

BizReachのAI Product Studioが開発した「Agentic UAT」は、AIエージェントがエンドユーザーを演じて業務をテストする手法。従来のテスト（UT・IT・E2E）が実装者視点であるのに対し、本手法は「ユーザーが実際に業務を完遂できるか」を検証する。

背景：AIが実装を担当する際に「テストは通るがユーザー視点では動かない」バグが発生しやすい課題がある。

## 技術的構成

### エージェント構造

```
UATマネージャー
  ├── ペルソナA
  ├── ペルソナB
  └── ペルソナC
```

### エージェントの制約条件

- ブラウザUI操作のみ（API直接呼び出し禁止）
- ソースコード読取禁止
- 仕様知識なしでUIから手順を発見
- 詰まり方そのものを記録

### ペルソナ設計

「担当業務」と「業務習熟度」の2軸。同じ画面でも、新担当者とベテランでは詰まるポイントが異なる。

## E2EテストとAgentic UATの根本的差異

**E2Eテスト**：答えを知っている人が手順を指示
```
1. /users/new にアクセス
2. 名前欄に入力
3. 保存ボタンをクリック
```

**Agentic UAT**：業務ゴールのみを伝える
```
新しいユーザーを登録してください。
登録後、ステータスを有効にしてください。
```

## 問題の分類

| 分類 | 定義 |
|------|------|
| Critical | 業務完遂不可 |
| Major | 完遂可だが大きな障壁 |
| Minor | 改善余地程度 |

## 制限事項

- 視覚的課題（レイアウトの微妙なおかしさ）は不得意
- ペルソナが実際の行動パターンを完全には再現できない
- 実行時間が数十分に及ぶため頻繁な実行には不向き
- 業務要件充足確認のみ（「使いにくさの直感」は検出不可）

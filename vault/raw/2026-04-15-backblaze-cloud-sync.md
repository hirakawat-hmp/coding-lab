# Raw: Backblaze Stops Backing Up OneDrive and Dropbox

Source: https://rareese.com/posts/backblaze/
Fetched: 2026-04-15

Backblaze quietly stopped backing up files stored in cloud storage services (OneDrive, Dropbox, Git folders) without notifying users.

## Key Details

- **Discovery**: User with 383GB OneDrive data found it was no longer being backed up
- **Official Justification**: Backblaze: exclusions prevent "performance issues, excessive data usage, and unintended uploads"
- **Hidden Exclusions**: Change buried in release notes as "Improvements" without direct user notification
- **Original Promise (2015)**: Website proclaimed "All user data included by default" with "No restrictions on file type or size"

## Impact

- Trust erosion after decade-long customer relationships damaged
- Cloud storage syncing ≠ backup: OneDrive/Dropbox have only 30-day retention vs. Backblaze's 1-year standard
- Users now question what other file types might be silently excluded

# Claude Code の3層設計パターン（CLAUDE.md / Skills / Agents）

source: https://qiita.com/ennagara128/items/c25e72eb240611454457
retrieved: 2026-05-02

## 概要
Claude Codeの設定ファイル肥大化問題に対する解決策として「3層分離設計パターン」を提案。

## 3層モデル

### 基盤層（CLAUDE.md）
- プロジェクト全体で常に適用すべきルール
- コーディング規約、禁止事項、命名規則など
- 200行以下に制限で可読性維持

### 手順層（Skills）
- 特定キーワードで呼び出される手順書
- デプロイやバグ調査などタスク固有の流れ
- 各Skillは単一目的、200〜300行が目安

### タスク層（Agents）
- 並列実行可能な独立したタスク単位
- コードレビューやドキュメント生成専用
- ステートレス設計により複数タスクの同時実行を実現

## 導入効果
- CLAUDE.mdが800行から180行へ80%削減
- Skillsの重複率が30%から5%に低下

## Key Facts

- PEP 832: standardizes .venv as default virtual environment directory
- Target: Python 3.15
- Status: Draft (as of April 17, 2026)

## Three Location Categories

1. Locally-created virtual environments per project
2. Named directory virtual environments (dot-prefixed versions hidden from tools)
3. Unified version handling across multiple Python versions

## Recommendations

- Projects should default to .venv subdirectory in project root
- When .venv not used, create mapping file for default virtual environment paths

## Community Reception

- Generated discussion on Python forums
- Some maintainers expressed lukewarm support
- Concerns about whether standardization will gain consensus

# Vibe coding and agentic engineering are getting closer than I'd like

Source: https://simonwillison.net/2026/May/6/vibe-coding-and-agentic-engineering/
Retrieved: 2026-05-07
Author: Simon Willison

## Core Argument

Simon Willison explores how two previously distinct approaches to AI-assisted programming have begun converging. Initially:
- "Vibe coding" = writing code without understanding it
- "Agentic engineering" = professional developers leveraging AI while maintaining quality standards

However, this boundary has become uncomfortably blurred.

## Key Insights

### The Accountability Problem
Willison notes he's stopped reviewing every line of code AI agents generate for production systems. While AI tools like Claude Code consistently produce reliable code, this creates ethical tension: "if I haven't reviewed the code, is it really responsible for me to use this in production?"

He resolves this by treating AI agents like trusted teams at larger organizations—trusting their outputs until problems emerge.

### Evaluation Challenges
A major shift: repositories now look identical whether they represent careful work or rapid AI generation. This makes assessing software quality visually impossible. Willison now values actual usage over polished documentation as a reliability indicator.

### Structural Disruption
Productivity increases (200 to 2,000 lines daily) have exposed design bottlenecks upstream. The entire development lifecycle was architected around slower code production, requiring reimagined workflows.

### Career Resilience
Willison remains confident in professional developers' futures, viewing AI tools as amplifiers requiring existing expertise to use effectively.

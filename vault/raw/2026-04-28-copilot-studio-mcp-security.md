# Copilot Studio の MCP 利用におけるセキュリティ対策

Source: Qiita https://qiita.com/Takashi_Masumori/items/5045c0d46a756f0a49fd
Fetched: 2026-04-28

## 2つのMCPタイプとその対策

### 1. 事前構築済みMicrosoft MCPコネクタ

**特徴**
従来のPower Platformコネクタと同様に扱われる。「Work IQ」など新しいコネクタが頻繁に追加。

**リスク**
アップデート速度が速いため、意図しないコネクタへのアクセスが発生する可能性がある。

**推奨される軽減策**
- DLPポリシーで制御
- Copilot Studio環境専用のポリシー見直しサイクルを短縮化
- 新規追加コネクタを「ブロック済み」または「非ビジネス」グループに設定
- 検証後に必要なコネクタのみビジネスグループへ移行

### 2. カスタムMCPサーバー

**技術的詳細**
内部的にはカスタムコネクタとして動作するため、「カスタムコネクタ側の DLP 設定で制御可能」。

**リスク**
未承認の外部MCPサーバーへの接続が懸念される。

**推奨される軽減策**
- カスタムコネクタと同じ審査運用プロセスを適用
- デフォルトで全カスタムMCPサーバーをブロック
- 個別申請時にURL単位での許可付与
- 必要に応じて環境分離を実施

## 実装上の考慮事項

「既存のカスタムコネクタ審査運用をそのまま流用できる」ため、新たなセキュリティフレームワーク構築は不要。ただしCopilot Studio専用環境では、DLPポリシーの見直し頻度の増加に対応する体制整備が重要。

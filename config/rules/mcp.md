# MCP Tools

MCP ツールの使い分けガイド。

## serena — セマンティックコード解析

コードベースの構造を理解する際に使う。

- `get_symbols_overview` でファイル内のシンボル一覧を取得
- `find_symbol` で特定のクラス・関数を検索（`include_body=True` で本文も取得）
- `find_referencing_symbols` で参照元を追跡
- リファクタリング時は `rename_symbol`, `replace_symbol_body` を活用

**使うタイミング:** シンボル探索、リファクタリング、大規模コードベースの構造把握

## context7 — ライブラリドキュメント参照（プラグイン）

ライブラリやフレームワークの公式ドキュメントを取得する。context7-plugin として導入済み。

- **docs-researcher agent を優先**: メインコンテキストを汚さずドキュメントを検索できる
- MCP ツール直接呼び出し（`resolve-library-id` → `query-docs`）も可能だが、agent 経由が推奨
- `/context7:docs <ライブラリ名>` で手動検索もできる
- ライブラリの話題が出ると自動トリガーされるスキルも同梱

**使うタイミング:** API 構文、設定方法、バージョン移行、ライブラリ固有のデバッグ。
よく知っている技術でも使うこと — 訓練データが最新でない可能性がある。

**使わない:** リファクタリング、スクリプト作成、ビジネスロジックのデバッグ、一般的なプログラミング概念

## sequential-thinking — 段階的思考

複雑な問題を分解して段階的に考える。

**使うタイミング:** 設計判断、複雑なバグの原因分析、トレードオフの整理
